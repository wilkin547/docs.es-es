 <span data-ttu-id="6854e-108">Cuando un dominio de aplicación se configura para realizar instantáneas de los archivos, los ensamblados de la ruta de acceso de la aplicación se copian a otra ubicación y se cargan desde esa ubicación.</span><span class="sxs-lookup"><span data-stu-id="6854e-108">When an application domain is configured to shadow copy files, assemblies from the application path are copied to another location and loaded from that location.</span></span> <span data-ttu-id="6854e-109">La instantánea está bloqueada, pero el archivo de ensamblado original está desbloqueado y se puede actualizar.</span><span class="sxs-lookup"><span data-stu-id="6854e-109">The copy is locked, but the original assembly file is unlocked and can be updated.</span></span>

     <span data-ttu-id="6854e-122">La ruta de acceso base de la ubicación se forma concatenando la propiedad <xref:System.AppDomainSetup.ApplicationName%2A> a la propiedad <xref:System.AppDomainSetup.CachePath%2A> como un subdirectorio.</span><span class="sxs-lookup"><span data-stu-id="6854e-122">The base path for the location is formed by concatenating the <xref:System.AppDomainSetup.ApplicationName%2A> property to the <xref:System.AppDomainSetup.CachePath%2A> property as a subdirectory.</span></span> <span data-ttu-id="6854e-123">Las instantáneas de los ensamblados se colocan en los subdirectorios de esta ruta, no en la ruta de acceso base.</span><span class="sxs-lookup"><span data-stu-id="6854e-123">Assemblies are shadow copied to subdirectories of this path, not to the base path itself.</span></span>  
  
    > [!NOTE]

## <a name="startup-performance"></a><span data-ttu-id="6854e-140">Rendimiento de inicio</span><span class="sxs-lookup"><span data-stu-id="6854e-140">Startup Performance</span></span>  
 <span data-ttu-id="6854e-141">Cuando se inicia un dominio de aplicación que utiliza instantáneas, hay un retraso mientras las instantáneas del directorio de aplicación se copian al directorio de instantáneas, o se comprueba si ya están en esa ubicación.</span><span class="sxs-lookup"><span data-stu-id="6854e-141">When an application domain that uses shadow copying starts, there is a delay while assemblies in the application directory are copied to the shadow copy directory, or verified if they are already in that location.</span></span> <span data-ttu-id="6854e-142">Antes de [!INCLUDE[net_v40_short](../../../includes/net-v40-short-md.md)], todos los ensamblados se copiaban a un directorio temporal.</span><span class="sxs-lookup"><span data-stu-id="6854e-142">Before the [!INCLUDE[net_v40_short](../../../includes/net-v40-short-md.md)], all assemblies were copied to a temporary directory.</span></span> <span data-ttu-id="6854e-143">Cada ensamblado se abría para comprobar el nombre del ensamblado y el nombre seguro se validaba.</span><span class="sxs-lookup"><span data-stu-id="6854e-143">Each assembly was opened to verify the assembly name, and the strong name was validated.</span></span> <span data-ttu-id="6854e-144">Se comprobaba cada ensamblado para saber si se actualizó más recientemente que la copia del directorio de instantáneas.</span><span class="sxs-lookup"><span data-stu-id="6854e-144">Each assembly was checked to see whether it had been updated more recently than the copy in the shadow copy directory.</span></span> <span data-ttu-id="6854e-145">Si es así, se copiaba al directorio de instantáneas.</span><span class="sxs-lookup"><span data-stu-id="6854e-145">If so, it was copied to the shadow copy directory.</span></span> <span data-ttu-id="6854e-146">Por último, se descartaban las copias temporales.</span><span class="sxs-lookup"><span data-stu-id="6854e-146">Finally, the temporary copies were discarded.</span></span>  
  
 <span data-ttu-id="6854e-147">A partir de [!INCLUDE[net_v40_short](../../../includes/net-v40-short-md.md)], el comportamiento de inicio predeterminado consiste en comparar directamente la fecha y la hora de cada archivo de ensamblado del directorio de aplicación con la fecha y la hora de la copia situada en el directorio de instantáneas.</span><span class="sxs-lookup"><span data-stu-id="6854e-147">Beginning with the [!INCLUDE[net_v40_short](../../../includes/net-v40-short-md.md)], the default startup behavior is to directly compare the file date and time of each assembly in the application directory with the file date and time of the copy in the shadow copy directory.</span></span> <span data-ttu-id="6854e-148">Si el ensamblado se ha actualizado, se copiará con el mismo procedimiento que en las versiones anteriores de .NET Framework. Si no, se cargará la copia del directorio de instantáneas.</span><span class="sxs-lookup"><span data-stu-id="6854e-148">If the assembly has been updated, it is copied by using the same procedure as in earlier versions of the .NET Framework; otherwise, the copy in the shadow copy directory is loaded.</span></span>  
  
 <span data-ttu-id="6854e-149">La mejora del rendimiento resultante es mayor en las aplicaciones en las que los ensamblados no cambian con frecuencia y, normalmente, los cambios se producen en un pequeño subconjunto de ensamblados.</span><span class="sxs-lookup"><span data-stu-id="6854e-149">The resulting performance improvement is largest for applications in which assemblies do not change frequently and changes usually occur in a small subset of assemblies.</span></span> <span data-ttu-id="6854e-150">Si la mayoría de los ensamblados de una aplicación cambia con frecuencia, el nuevo comportamiento predeterminado puede provocar una regresión del rendimiento.</span><span class="sxs-lookup"><span data-stu-id="6854e-150">If a majority of assemblies in an application change frequently, the new default behavior might cause a performance regression.</span></span> <span data-ttu-id="6854e-151">Puede restaurar el comportamiento de inicio de las versiones anteriores de .NET Framework.
