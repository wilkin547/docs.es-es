 <span data-ttu-id="c204d-116">La clase <xref:System.Uri?displayProperty=nameWithType> existente se ha ampliado para proporcionar compatibilidad con IRI basada en RFC 3987.</span><span class="sxs-lookup"><span data-stu-id="c204d-116">The existing <xref:System.Uri?displayProperty=nameWithType> class has been extended to provide IRI support based on RFC 3987.</span></span> <span data-ttu-id="c204d-117">Los usuarios actuales no percibirán ningún cambio respecto al comportamiento de .NET Framework 2.0, a menos que habiliten IRI específicamente.</span><span class="sxs-lookup"><span data-stu-id="c204d-117">Current users will not see any change from the .NET Framework 2.0 behavior unless they specifically enable IRI.</span></span> <span data-ttu-id="c204d-118">Esto garantiza la compatibilidad de las aplicaciones con versiones anteriores de .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="c204d-118">This ensures application compatibility with prior versions of the .NET Framework.</span></span>  
  
 <span data-ttu-id="c204d-119">Una aplicación puede especificar si se debe usar el análisis del Nombre de dominio internacionalizado (IDN) aplicado a nombres de dominio y si deben aplicarse reglas de análisis de IRI.</span><span class="sxs-lookup"><span data-stu-id="c204d-119">An application can specify whether to use Internationalized Domain Name (IDN) parsing applied to domain names and whether IRI parsing rules should be applied.</span></span> <span data-ttu-id="c204d-120">Esto puede hacerse en el archivo machine.config o app.config.</span><span class="sxs-lookup"><span data-stu-id="c204d-120">This can be done in the machine.config or in the app.config file.</span></span>
