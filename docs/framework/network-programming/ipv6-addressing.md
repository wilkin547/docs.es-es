# <a name="ipv6-addressing"></a><span data-ttu-id="4522c-102">Direccionamiento IPv6</span><span class="sxs-lookup"><span data-stu-id="4522c-102">IPv6 Addressing</span></span>
<span data-ttu-id="4522c-103">En el protocolo de Internet versión 6 (IPv6), las direcciones tienen una longitud de 128 bits.</span><span class="sxs-lookup"><span data-stu-id="4522c-103">In the Internet Protocol version 6 (IPv6), addresses are 128 bits long.</span></span> <span data-ttu-id="4522c-104">Entre otras razones, el espacio de direcciones es tan grande para subdividir las direcciones disponibles en una jerarquía de dominios de enrutamiento que reflejen la topología de Internet</span><span class="sxs-lookup"><span data-stu-id="4522c-104">One reason for such a large address space is to subdivide the available addresses into a hierarchy of routing domains that reflect the Internet's topology.</span></span> <span data-ttu-id="4522c-105">y para asignar las direcciones de los adaptadores de red (o interfaces) que conectan los dispositivos a la red.</span><span class="sxs-lookup"><span data-stu-id="4522c-105">Another reason is to map the addresses of network adapters (or interfaces) that connect devices to the network.</span></span> <span data-ttu-id="4522c-106">IPv6 se caracteriza por una capacidad inherente para resolver direcciones en su nivel más bajo, que es el nivel de la interfaz de red, y cuenta con funciones de configuración automática.</span><span class="sxs-lookup"><span data-stu-id="4522c-106">IPv6 features an inherent capability to resolve addresses at their lowest level, which is at the network interface level, and also has auto-configuration capabilities.</span></span>

-   <span data-ttu-id="4522c-113">**Formato comprimido**.</span><span class="sxs-lookup"><span data-stu-id="4522c-113">**Compressed form**.</span></span> <span data-ttu-id="4522c-114">Debido a la ## <a name="address-types"></a><span data-ttu-id="4522c-124">Tipos de dirección</span><span class="sxs-lookup"><span data-stu-id="4522c-124">Address Types</span></span>  
 <span data-ttu-id="4522c-125">Los bits iniciales de la dirección definen el tipo de dirección IPv6 específico.</span><span class="sxs-lookup"><span data-stu-id="4522c-125">The leading bits in the address define the specific IPv6 address type.</span></span> <span data-ttu-id="4522c-126">El campo de longitud variable que contiene estos bits iniciales se denomina prefijo de formato (FP).</span><span class="sxs-lookup"><span data-stu-id="4522c-126">The variable-length field containing these leading bits is called a Format Prefix (FP).</span></span>  
  
 <span data-ttu-id="4522c-127">Una dirección de unidifusión de IPv6 se divide en dos partes.</span><span class="sxs-lookup"><span data-stu-id="4522c-127">An IPv6 unicast address is divided into two parts.</span></span> <span data-ttu-id="4522c-128">La primera parte contiene el prefijo de dirección y la segunda parte contiene el identificador de interfaz.</span><span class="sxs-lookup"><span data-stu-id="4522c-128">The first part contains the address prefix, and the second part contains the interface identifier.</span></span> <span data-ttu-id="4522c-129">Una forma concisa de expresar una combinación de dirección y prefijo IPv6 es la siguiente: dirección ipv6/longitud de prefijo.</span><span class="sxs-lookup"><span data-stu-id="4522c-129">A concise way to express an IPv6 address/prefix combination is as follows: ipv6-address/prefix-length.</span></span>  
  
 <span data-ttu-id="4522c-130">A continuación se muestra un ejemplo de una dirección con un prefijo de 64 bits.</span><span class="sxs-lookup"><span data-stu-id="4522c-130">The following is an example of an address with a 64-bit prefix.</span></span>  
  
 <span data-ttu-id="4522c-131">`3FFE:FFFF:0:CD30:0:0:0:0/64`.</span><span class="sxs-lookup"><span data-stu-id="4522c-131">`3FFE:FFFF:0:CD30:0:0:0:0/64`.</span></span>  
  
 <span data-ttu-id="4522c-132">El prefijo de este ejemplo es `3FFE:FFFF:0:CD30`.</span><span class="sxs-lookup"><span data-stu-id="4522c-132">The prefix in this example is `3FFE:FFFF:0:CD30`.</span></span> <span data-ttu-id="4522c-133">La dirección también se puede escribir en formato comprimido, como `3FFE:FFFF:0:CD30::/64`.</span><span class="sxs-lookup"><span data-stu-id="4522c-133">The address can also be written in a compressed form, as `3FFE:FFFF:0:CD30::/64`.</span></span>  
  
 <span data-ttu-id="4522c-134">IPv6 define los siguientes tipos de dirección:</span><span class="sxs-lookup"><span data-stu-id="4522c-134">IPv6 defines the following address types:</span></span>  
  
-   <span data-ttu-id="4522c-135">**Dirección de unidifusión**.</span><span class="sxs-lookup"><span data-stu-id="4522c-135">**Unicast address**.</span></span> <span data-ttu-id="4522c-136">Identificador de una única interfaz.</span><span class="sxs-lookup"><span data-stu-id="4522c-136">An identifier for a single interface.</span></span> <span data-ttu-id="4522c-137">Un paquete enviado a esta dirección se entrega a la interfaz identificada.</span><span class="sxs-lookup"><span data-stu-id="4522c-137">A packet sent to this address is delivered to the identified interface.</span></span> <span data-ttu-id="4522c-138">Las direcciones de unidifusión se diferencian de las direcciones de multidifusión por el valor del octeto de orden superior.</span><span class="sxs-lookup"><span data-stu-id="4522c-138">The unicast addresses are distinguished from the multicast addresses by the value of the high-order octet.</span></span> <span data-ttu-id="4522c-139">El octeto de orden superior de las direcciones de multidifusión tiene el valor hexadecimal de FF.</span><span class="sxs-lookup"><span data-stu-id="4522c-139">The multicast addresses' high-order octet has the hexadecimal value of FF.</span></span> <span data-ttu-id="4522c-140">Los demás valores de este octeto identifican una dirección de unidifusión.</span><span class="sxs-lookup"><span data-stu-id="4522c-140">Any other value for this octet identifies a unicast address.</span></span> <span data-ttu-id="4522c-141">A continuación se muestran diferentes tipos de direcciones de unidifusión:</span><span class="sxs-lookup"><span data-stu-id="4522c-141">The following are different types of unicast addresses:</span></span>  
  
    -   <span data-ttu-id="4522c-142">**Direcciones locales de vínculo**.</span><span class="sxs-lookup"><span data-stu-id="4522c-142">**Link-local addresses**.</span></span> <span data-ttu-id="4522c-143">Estas direcciones se usan en un único vínculo y tienen el formato siguiente: FE80::*InterfaceID*.</span><span class="sxs-lookup"><span data-stu-id="4522c-143">These addresses are used on a single link and have the following format: FE80::*InterfaceID*.</span></span> <span data-ttu-id="4522c-144">Las direcciones locales de vínculo se usan entre nodos en un vínculo para la configuración de dirección automática, la detección de equipos cercanos o cuando no hay enrutadores presentes.</span><span class="sxs-lookup"><span data-stu-id="4522c-144">Link-local addresses are used between nodes on a link for auto-address configuration, neighbor discovery, or when no routers are present.</span></span> <span data-ttu-id="4522c-145">Las direcciones locales de vínculo se usan principalmente en el inicio y cuando el sistema todavía no ha adquirido direcciones de ámbito mayor.</span><span class="sxs-lookup"><span data-stu-id="4522c-145">A link-local address is used primarily at startup and when the system has not yet acquired addresses of larger scope.</span></span>

-   <span data-ttu-id="4522c-155">**Dirección de difusión por proximidad**.</span><span class="sxs-lookup"><span data-stu-id="4522c-155">**Anycast address**.</span></span> <span data-ttu-id="4522c-156">Identificador para un conjunto de interfaces (normalmente pertenecientes a diferentes nodos).</span><span class="sxs-lookup"><span data-stu-id="4522c-156">An identifier for a set of interfaces (typically belonging to different nodes).</span></span> <span data-ttu-id="4522c-157">Un paquete enviado a esta dirección se entrega a una sola interfaz identificada por la dirección.</span><span class="sxs-lookup"><span data-stu-id="4522c-157">A packet sent to this address is delivered to only one interface identified by the address.</span></span> <span data-ttu-id="4522c-158">Se trata de la interfaz más próxima identificada por la métrica de enrutamiento.</span><span class="sxs-lookup"><span data-stu-id="4522c-158">This is the nearest interface as identified by routing metrics.</span></span> <span data-ttu-id="4522c-159">Las direcciones de difusión por proximidad se toman del espacio de direcciones de unidifusión y no se pueden distinguir sintácticamente.</span><span class="sxs-lookup"><span data-stu-id="4522c-159">Anycast addresses are taken from the unicast address space and are not syntactically distinguishable.</span></span> <span data-ttu-id="4522c-160">La interfaz a la que se realiza el direccionamiento realiza la distinción entre las direcciones de unidifusión y de difusión por proximidad como una función de su configuración.</span><span class="sxs-lookup"><span data-stu-id="4522c-160">The addressed interface performs the distinction between unicast and anycast addresses as a function of its configuration.</span></span>
