<span data-ttu-id="042ff-108">Cuando se compila en .NET Framework código administrado, los compiladores como Visual C++, Visual Basic y C# compilan el programa fuente en lenguaje intermedio de Microsoft (MSIL).</span><span class="sxs-lookup"><span data-stu-id="042ff-108">When compiling .NET Framework managed code, compilers such as Visual C++, Visual Basic, and C# compile their source program into Microsoft intermediate language (MSIL).</span></span> <span data-ttu-id="042ff-109">MSIL es, a continuación, compila con JIT, justo antes de ejecutarse, en código máquina nativo.</span><span class="sxs-lookup"><span data-stu-id="042ff-109">MSIL is then JIT-compiled, just before execution, into native machine code.</span></span> <span data-ttu-id="042ff-110">Al igual que con el código no administrado, puede configurar una imagen ejecutable para depurarla si establece algunos modificadores IDE u opciones de línea de comandos.</span><span class="sxs-lookup"><span data-stu-id="042ff-110">As with unmanaged code, you can configure an executable image for debugging by setting IDE switches or command-line options.</span></span> <span data-ttu-id="042ff-111">También puede configurar la compilación JIT para la depuración de la misma manera.</span><span class="sxs-lookup"><span data-stu-id="042ff-111">You can also configure the JIT compilation for debugging in much the same way.</span></span>

<span data-ttu-id="042ff-112">Esta configuración JIT tiene dos aspectos:</span><span class="sxs-lookup"><span data-stu-id="042ff-112">This JIT configuration has two aspects:</span></span>

- <span data-ttu-id="042ff-113">Puede solicitar el compilador JIT para generar información de seguimiento.</span><span class="sxs-lookup"><span data-stu-id="042ff-113">You can request the JIT compiler to generate tracking information.</span></span> <span data-ttu-id="042ff-114">Esto permite que el depurador empareje una cadena de MSIL con su equivalente de código administrado y realizar un seguimiento de dónde se almacenan las variables locales y los argumentos de función.</span><span class="sxs-lookup"><span data-stu-id="042ff-114">This makes it possible for the debugger to match up a chain of MSIL with its machine code counterpart, and to track where local variables and function arguments are stored.</span></span> <span data-ttu-id="042ff-115">A partir de la versión 2.0 de .NET Framework, el compilador JIT genera siempre información de seguimiento, así que no hay ninguna necesidad de solicitarla.</span><span class="sxs-lookup"><span data-stu-id="042ff-115">Starting with the .NET Framework version 2.0, the JIT compiler always generates tracking information, so there is no need to request it.</span></span>

<span data-ttu-id="042ff-117">Normalmente, el compilador que genera el MSIL establece estas opciones del compilador JIT la forma adecuada, en función de los modificadores IDE u opciones de línea de comandos que especifique, por ejemplo, /**Od**.</span><span class="sxs-lookup"><span data-stu-id="042ff-117">Normally, the compiler that generates the MSIL sets these JIT compiler options appropriately, based upon the IDE switches or command-line options you specify, for example, /**Od**.</span></span>

<span data-ttu-id="042ff-121">Por ejemplo, si el ensamblado que desea depurar se llama a *MyApp.exe*, a continuación, puede crear un archivo de texto denominado *MyApp.ini*, en la misma carpeta que *MyApp.exe*, que contiene estas tres líneas:</span><span class="sxs-lookup"><span data-stu-id="042ff-121">For example, if the assembly you want to debug is called *MyApp.exe*, then you can create a text file named *MyApp.ini*, in the same folder as *MyApp.exe*, which contains these three lines:</span></span>

<span data-ttu-id="042ff-122">Puede establecer el valor de cada opción en 0 o 1. <span data-ttu-id="042ff-124">A partir de la versión 2.0 de .NET Framework, el compilador JIT genera siempre información de seguimiento, independientemente del valor de `GenerateTrackingInfo`; sin embargo, el `AllowOptimize` valor sigue teniendo efecto.</span><span class="sxs-lookup"><span data-stu-id="042ff-124">Starting with the .NET Framework version 2.0, the JIT compiler always generates tracking information regardless of the value for `GenerateTrackingInfo`; <span data-ttu-id="042ff-128"><xref:System.Diagnostics.DebuggableAttribute?displayProperty=nameWithType> controla los valores de un ensamblado.</span><span class="sxs-lookup"><span data-stu-id="042ff-128">The <xref:System.Diagnostics.DebuggableAttribute?displayProperty=nameWithType> controls the settings for an assembly.</span></span> <span data-ttu-id="042ff-129">**DebuggableAttribute** incluye dos campos que controlan si el compilador JIT debe optimizar y/o generar información de seguimiento.</span><span class="sxs-lookup"><span data-stu-id="042ff-129">**DebuggableAttribute** includes two fields that control whether the JIT compiler should optimize and/or generate tracking information.</span></span> <span data-ttu-id="042ff-130">A partir de la versión 2.0 de .NET Framework, el compilador JIT genera siempre información de seguimiento.</span><span class="sxs-lookup"><span data-stu-id="042ff-130">Starting with the .NET Framework version 2.0, the JIT compiler always generates tracking information.</span></span>
