 <span data-ttu-id="2768f-106">La mensajería de confianza se define como la manera en que un origen de mensajería de confianza (denominado origen) transfiere mensajes de manera fiable a un destino de mensajería de confianza (denominado destino).</span><span class="sxs-lookup"><span data-stu-id="2768f-106">Reliable messaging is how a reliable messaging source (called the source) transfers messages reliably to a reliable messaging destination (called the destination).</span></span>  
  
 <span data-ttu-id="2768f-107">La mensajería de confianza tiene los siguientes aspectos clave:</span><span class="sxs-lookup"><span data-stu-id="2768f-107">Reliable messaging has the following key aspects:</span></span>

 <span data-ttu-id="2768f-110">La mensajería de confianza a menudo viene acompañada de una alta latencia.</span><span class="sxs-lookup"><span data-stu-id="2768f-110">Reliable messaging frequently comes at the cost of high latency.</span></span> <span data-ttu-id="2768f-111">La latencia es el tiempo que tarda un mensaje en alcanzar el destino desde el origen.</span><span class="sxs-lookup"><span data-stu-id="2768f-111">Latency is the time it takes for the message to reach the destination from the source.</span></span> <span data-ttu-id="2768f-112">WCF, por lo tanto, proporciona los siguientes tipos de mensajería de confianza:</span><span class="sxs-lookup"><span data-stu-id="2768f-112">WCF, therefore, provides the following types of reliable messaging:</span></span>  
  
- <span data-ttu-id="2768f-113">[Sesiones confiables](reliable-sessions.md), que ofrecen transferencia confiable sin el costo de latencia alta</span><span class="sxs-lookup"><span data-stu-id="2768f-113">[Reliable Sessions](reliable-sessions.md), which offer reliable transfer without the cost of high latency</span></span>  
  
- <span data-ttu-id="2768f-114">[Colas en WCF](queues-in-wcf.md), que ofrecen transferencias de confianza y separación entre el origen y el destino.</span><span class="sxs-lookup"><span data-stu-id="2768f-114">[Queues in WCF](queues-in-wcf.md), which offer both reliable transfers and separation between the source and the destination.</span></span>  
  
## <a name="reliable-sessions"></a><span data-ttu-id="2768f-115">Sesiones de confianza</span><span class="sxs-lookup"><span data-stu-id="2768f-115">Reliable Sessions</span></span>  

 <span data-ttu-id="2768f-116">Las sesiones de confianza proporcionan una transferencia confiable de un punto de conexión a otro de mensajes entre un origen y un destino mediante el protocolo WS-ReliableMessaging, sin tener en cuenta el número o tipo de intermediarios que separan los puntos de conexión de la mensajería (origen y destino).</span><span class="sxs-lookup"><span data-stu-id="2768f-116">Reliable sessions provide end-to-end reliable transfer of messages between a source and a destination using the WS-ReliableMessaging protocol regardless of the number or type of intermediaries that separate the messaging (source and destination) endpoints.</span></span> <span data-ttu-id="2768f-117">Esto incluye a cualquier intermediario de transporte que no utilice SOAP (por ejemplo, los servidores proxy HTTP) o los intermediarios que utilicen SOAP (por ejemplo, los puentes o enrutadores basados en SOAP) que son necesarios para que los mensajes fluyan entre los extremos.</span><span class="sxs-lookup"><span data-stu-id="2768f-117">This includes any transport intermediaries that do not use SOAP (for example, HTTP proxies) or intermediaries that use SOAP (for example, SOAP-based routers or bridges) that are required for messages to flow between the endpoints.</span></span> <span data-ttu-id="2768f-118">Las sesiones confiables utilizan una ventana de transferencia en memoria para enmascarar errores de nivel de mensaje de SOAP y restablecer las conexiones en el caso de errores de transporte.</span><span class="sxs-lookup"><span data-stu-id="2768f-118">Reliable sessions use an in-memory transfer window to mask SOAP message-level failures and re-establish connections in the case of transport failures.</span></span>  
  
 <span data-ttu-id="2768f-119">Las sesiones de confianza proporcionan transferencias de mensajes de confianza de latencia baja.</span><span class="sxs-lookup"><span data-stu-id="2768f-119">Reliable sessions provide low-latency reliable message transfers.</span></span> <span data-ttu-id="2768f-120">Los proporcionan para los mensajes SOAP sobre cualquier proxy o intermediario, el equivalente a lo que TCP proporciona para los paquetes sobre puentes de IP.</span><span class="sxs-lookup"><span data-stu-id="2768f-120">They provide for SOAP messages over any proxies or intermediaries, equivalent to what TCP provides for packets over IP bridges.</span></span> <span data-ttu-id="2768f-121">Para obtener más información acerca de las sesiones confiables, vea [sesiones confiables](reliable-sessions.md).</span><span class="sxs-lookup"><span data-stu-id="2768f-121">For more information about reliable sessions, see [Reliable Sessions](reliable-sessions.md).</span></span>  
  
## <a name="queues"></a><span data-ttu-id="2768f-122">Colas</span><span class="sxs-lookup"><span data-stu-id="2768f-122">Queues</span></span>  

 <span data-ttu-id="2768f-123">Las colas en WCF proporcionan transferencias confiables de mensajes y separación entre orígenes y destinos a costa de una latencia alta.</span><span class="sxs-lookup"><span data-stu-id="2768f-123">Queues in WCF provide both reliable transfers of messages and separation between sources and destinations at the cost of high latency.</span></span> <span data-ttu-id="2768f-124">La comunicación en cola de WCF se basa en Message Queue Server (también conocido como MSMQ).</span><span class="sxs-lookup"><span data-stu-id="2768f-124">WCF queued communication is built on top of Message Queuing (also known as MSMQ).</span></span>  
  
 <span data-ttu-id="2768f-125">MSMQ se distribuye como una opción con Windows que se ejecuta como un servicio de NT.</span><span class="sxs-lookup"><span data-stu-id="2768f-125">MSMQ is shipped as an option with Windows that runs as an NT service.</span></span> <span data-ttu-id="2768f-126">Captura mensajes para la transmisión en una cola de transmisión en nombre del origen y lo entrega a una cola de destino.</span><span class="sxs-lookup"><span data-stu-id="2768f-126">It captures messages for transmission in a transmission queue on behalf of the source and delivers it to a target queue.</span></span> <span data-ttu-id="2768f-127">La cola de destino acepta los mensajes en nombre del destino para la entrega posterior siempre que el destino solicite mensajes.</span><span class="sxs-lookup"><span data-stu-id="2768f-127">The target queue accepts messages on behalf of the destination for later delivery whenever the destination requests for messages.</span></span> <span data-ttu-id="2768f-128">Los administradores de cola de MSMQ implementan un protocolo de transferencias de mensajes de confianza para que los mensajes no se pierdan durante la transmisión.</span><span class="sxs-lookup"><span data-stu-id="2768f-128">The MSMQ queue managers implement a reliable message-transfer protocol so that messages are not lost in transmission.</span></span> <span data-ttu-id="2768f-129">El protocolo puede ser nativo o basado en SOAP, como, por ejemplo, Soap Reliable Messaging Protocol (SRMP).</span><span class="sxs-lookup"><span data-stu-id="2768f-129">The protocol can be native or SOAP-based, such as Soap Reliable Messaging Protocol (SRMP).</span></span>  
  
 <span data-ttu-id="2768f-130">La separación, acoplada con las transferencias de mensaje de confianza entre colas, permite que las aplicaciones que están acopladas se comuniquen de forma fiable.</span><span class="sxs-lookup"><span data-stu-id="2768f-130">The separation, coupled with reliable message transfers between queues, enables applications that are loosely coupled to communicate reliably.</span></span> <span data-ttu-id="2768f-131">A diferencia de las sesiones de confianza, el origen y el destino no tienen que ejecutarse a la vez.</span><span class="sxs-lookup"><span data-stu-id="2768f-131">Unlike reliable sessions, the source and destination do not have to be running at the same time.</span></span> <span data-ttu-id="2768f-132">Esto habilita escenarios de forma implícita allí donde se usan las colas como mecanismo de nivelación de carga cuando hay una discordancia entre la velocidad de producción de mensajes por parte del origen y la velocidad de uso de mensajes por parte del destino.</span><span class="sxs-lookup"><span data-stu-id="2768f-132">This implicitly enables scenarios where queues are, in effect, used as a load-leveling mechanism when there is a mismatch between the rate of message production by the source and the rate of the message consumption by the destination.</span></span> <span data-ttu-id="2768f-133">Para obtener más información acerca de las colas, vea [colas en WCF](queues-in-wcf.md).</span><span class="sxs-lookup"><span data-stu-id="2768f-133">For more information about queues, see [Queues in WCF](queues-in-wcf.md).</span></span>
