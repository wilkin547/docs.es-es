# <a name="adopt-windows-communication-foundation"></a><span data-ttu-id="1e1a5-103">Adopción de Windows Communication Foundation</span><span class="sxs-lookup"><span data-stu-id="1e1a5-103">Adopt Windows Communication Foundation</span></span>

<span data-ttu-id="1e1a5-104">Puede optar por usar Windows Communication Foundation (WCF) para el nuevo desarrollo mientras continúa manteniendo las aplicaciones existentes desarrolladas mediante ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-104">You can choose to use Windows Communication Foundation (WCF) for new development while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="1e1a5-105">Dado que WCF pretende ser la opción más adecuada para facilitar la comunicación con las aplicaciones compiladas con la .NET Framework en cualquier escenario, puede actuar como una herramienta estándar para resolver una gran variedad de problemas de comunicación de software de una forma que ASP.NET no pueda.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-105">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>

<span data-ttu-id="1e1a5-106">Las nuevas aplicaciones WCF se pueden implementar en los mismos equipos que los servicios Web de ASP.NET existentes.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-106">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="1e1a5-107">Si los servicios Web de ASP.NET existentes usan una versión de la .NET Framework anterior a la 2,0, puede usar la herramienta de registro de IIS de ASP.NET para implementar de forma selectiva el .NET Framework 2,0 en las aplicaciones de IIS en las que se van a hospedar nuevas aplicaciones WCF.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-107">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="1e1a5-108">Esa herramienta está documentada en [ASP.net IIS registration Tool (Aspnet_regiis.exe)](/previous-versions/dotnet/netframework-3.5/k6h9cz8h(v=vs.90))y tiene una interfaz de usuario integrada en la consola de administración de IIS 6,0.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-108">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](/previous-versions/dotnet/netframework-3.5/k6h9cz8h(v=vs.90)), and has a user interface built into the IIS 6.0 management console.</span></span>

<span data-ttu-id="1e1a5-109">WCF se puede usar para agregar nuevas características a los servicios Web de ASP.NET existentes agregando servicios WCF configurados para ejecutarse en el modo de compatibilidad de ASP.NET a las aplicaciones de servicio Web ASP.NET existentes en IIS.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-109">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="1e1a5-110">Debido al modo de compatibilidad de ASP.NET, el código de los nuevos servicios WCF puede obtener acceso y actualizar la misma información de estado de la aplicación que el código ASP.NET existente, mediante la <xref:System.Web.HttpContext> clase.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-110">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="1e1a5-111">Las aplicaciones también pueden compartir las mismas bibliotecas de clases.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-111">The applications can also share the same class libraries.</span></span>

<span data-ttu-id="1e1a5-112">Los clientes de WCF pueden usar los servicios Web de ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-112">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="1e1a5-113">Los servicios WCF que se configuran con <xref:System.ServiceModel.BasicHttpBinding> pueden ser usados por los clientes del servicio Web de ASP.net.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-113">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="1e1a5-114">Los servicios Web ASP.NET pueden coexistir con aplicaciones WCF y, incluso, se puede usar WCF para agregar características a los servicios Web ASP.NET existentes.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-114">ASP.NET Web services can coexist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="1e1a5-115">Teniendo en cuenta todos estos métodos en los que se pueden usar juntos los servicios Web WCF y ASP.NET, puede que desee migrar los servicios Web de ASP.NET a WCF solo si necesita características proporcionadas por WCF y no servicios Web de ASP.NET.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-115">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>

<span data-ttu-id="1e1a5-116">Incluso en los pocos casos en los que es necesario, la migración de código de una tecnología a otra rara vez es el enfoque correcto.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-116">Even in the few cases where it is necessary, migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="1e1a5-117">La razón para adoptar la nueva tecnología es cumplir los nuevos requisitos que no se pueden cumplir con la tecnología anterior y, en ese caso, lo correcto es diseñar una solución nueva para cumplir con el conjunto de requisitos recién expandido.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-117">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly expanded set of requirements.</span></span> <span data-ttu-id="1e1a5-118">El nuevo diseño se beneficia de su experiencia con el sistema existente y de la sabiduría adquirida desde que se diseñó el sistema.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-118">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="1e1a5-119">El nuevo diseño también puede utilizar las funciones completas de las nuevas tecnologías en lugar de reproducir el diseño anterior en la nueva plataforma.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-119">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="1e1a5-120">Después de crear prototipos de elementos clave del nuevo diseño, es más fácil reutilizar el código del sistema existente en el nuevo.</span><span class="sxs-lookup"><span data-stu-id="1e1a5-120">After prototyping key elements of the new design, it becomes easier to reuse code from the existing system within the new one.</span></span>
