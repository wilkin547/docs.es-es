# <a name="secure-conversations-and-secure-sessions"></a><span data-ttu-id="d1e8a-102">Conversaciones y sesiones seguras</span><span class="sxs-lookup"><span data-stu-id="d1e8a-102">Secure Conversations and Secure Sessions</span></span>
<span data-ttu-id="d1e8a-103">Una característica de Windows Communication Foundation (WCF) es la capacidad de establecer sesiones seguras entre dos puntos de conexión que se autentican entre sí y aceptan un cifrado y un proceso de firma digital.</span><span class="sxs-lookup"><span data-stu-id="d1e8a-103">A feature of Windows Communication Foundation (WCF) is the ability to establish secure sessions between two endpoints that authenticate each other and agree upon an encryption and digital signature process.</span></span> <span data-ttu-id="d1e8a-104">Por ejemplo, el punto de conexión de servicio podría exigir a un punto de conexión del cliente que envíe un token de seguridad basado en un certificado X.509 para su autenticación.</span><span class="sxs-lookup"><span data-stu-id="d1e8a-104">For example, the service endpoint might require a client endpoint to send a security token based upon an X.509 certificate for authentication.</span></span> <span data-ttu-id="d1e8a-105">Una vez autenticado el cliente, el extremo de servicio devuelve un token de contexto de seguridad (SCT) al cliente que se utiliza a continuación para proteger todos los mensajes subsiguientes dentro de la sesión.</span><span class="sxs-lookup"><span data-stu-id="d1e8a-105">Once the client is authenticated, the service endpoint returns a security context token (SCT) back to the client that is then used to secure all subsequent messages within the session.</span></span> <span data-ttu-id="d1e8a-106">Al establecer esta sesión segura, se habilita el conjunto de mensajes que se intercambian entre los dos extremos para ser más eficaz, porque el SCT tiene una clave simétrica.</span><span class="sxs-lookup"><span data-stu-id="d1e8a-106">Establishing this secure session enables the set of messages that are exchanged between the two endpoints to be more efficient, because the SCT has a symmetric key.</span></span> <span data-ttu-id="d1e8a-107">Las claves asimétricas, en las que se basan los certificados X.509, requieren una potencia de cálculo bastante mayor que las claves simétricas cuando se genera una firma digital o se cifra un conjunto de datos.</span><span class="sxs-lookup"><span data-stu-id="d1e8a-107">Asymmetric keys, which X.509 certificates are based upon, require significantly more computational power than symmetric keys when generating a digital signature or encrypting a set of data.</span></span>  
  
 <span data-ttu-id="d1e8a-108">La Directiva bootstrap (definida en la sección 6.2.7 del estándar [WS-SecurityPolicy](https://docs.oasis-open.org/ws-sx/ws-securitypolicy/200702/ws-securitypolicy-1.2-spec-os.html) ) contiene las aserciones de seguridad del mensaje que se usan para proteger el canal y autenticar el cliente antes del intercambio RST/SCT y RSTR/SCT.</span><span class="sxs-lookup"><span data-stu-id="d1e8a-108">The bootstrap policy (defined in section 6.2.7 of the [WS-SecurityPolicy](https://docs.oasis-open.org/ws-sx/ws-securitypolicy/200702/ws-securitypolicy-1.2-spec-os.html) standard) contains the message security assertions used to secure the channel and authenticate the client prior to the RST/SCT and RSTR/SCT exchange.</span></span> <span data-ttu-id="d1e8a-109">Algunos enlaces estándar de WCF tienen una propiedad `Security.Message.EstablishSecurityContext` que controla si se utiliza la conversación segura.</span><span class="sxs-lookup"><span data-stu-id="d1e8a-109">Certain WCF standard bindings have a `Security.Message.EstablishSecurityContext` property which controls whether secure conversation is used.</span></span> <span data-ttu-id="d1e8a-110">Al usar enlaces personalizados, el bootstrap se indica mediante el anidamiento de elementos de enlace de seguridad, ya sea a través de [\<> secureConversationBootstrap](../../../../docs/framework/configure-apps/file-schema/wcf/secureconversationbootstrap.md) en el archivo de configuración o mediante una llamada a <xref:System.ServiceModel.Channels.SecurityBindingElement.CreateSecureConversationBindingElement%2A> en el código.</span><span class="sxs-lookup"><span data-stu-id="d1e8a-110">When using custom bindings the bootstrap is indicated by nesting security binding elements, either through [\<secureConversationBootstrap>](../../../../docs/framework/configure-apps/file-schema/wcf/secureconversationbootstrap.md) in the configuration file, or by calling <xref:System.ServiceModel.Channels.SecurityBindingElement.CreateSecureConversationBindingElement%2A> in code.</span></span>
