## <a name="viewing-messages-logged-by-a-client-a-relay-and-a-service"></a><span data-ttu-id="f5ea2-117">Visualización de los mensajes registrados por un cliente, un relé y un servicio</span><span class="sxs-lookup"><span data-stu-id="f5ea2-117">Viewing Messages Logged by a Client, a Relay, and a Service</span></span>  
 <span data-ttu-id="f5ea2-118">Su entorno puede contener un cliente, que envía un mensaje a un relé, que seguidamente reenvía el mensaje al servicio.</span><span class="sxs-lookup"><span data-stu-id="f5ea2-118">Your environment may contain a client, which sends a message to a relay, that subsequently forwards the message to the service.</span></span> <span data-ttu-id="f5ea2-119">Cuando el registro de mensajes está habilitado en las tres ubicaciones y se ven todos los registros de mensaje tres en [herramienta Service Trace Viewer (SvcTraceViewer.exe)](../../../../docs/framework/wcf/service-trace-viewer-tool-svctraceviewer-exe.md) los intercambios de registro de mensajes al mismo tiempo, se representará correctamente.</span><span class="sxs-lookup"><span data-stu-id="f5ea2-119">When message logging is enabled on all three locations, and all three message logs are viewed in [Service Trace Viewer Tool (SvcTraceViewer.exe)](../../../../docs/framework/wcf/service-trace-viewer-tool-svctraceviewer-exe.md) simultaneously, the message log exchanges will be incorrectly rendered.</span></span> <span data-ttu-id="f5ea2-120">Esto se debe a que `CorrelationId` y `ActivityId` en el encabezado del mensaje no son únicos para cada par de envío-recepción.</span><span class="sxs-lookup"><span data-stu-id="f5ea2-120">This is because the `CorrelationId` and `ActivityId` in the Message header are not unique for every send-receive pair.</span></span>  
  
 <span data-ttu-id="f5ea2-121">Para solucionar este problema puede usar uno de los métodos siguientes:</span><span class="sxs-lookup"><span data-stu-id="f5ea2-121">You can use either one of the following methods to resolve this problem.</span></span>  
  
-   <span data-ttu-id="f5ea2-122">Ver sólo dos de los tres registros de mensajes en el [herramienta Service Trace Viewer (SvcTraceViewer.exe)](../../../../docs/framework/wcf/service-trace-viewer-tool-svctraceviewer-exe.md) en cualquier momento.</span><span class="sxs-lookup"><span data-stu-id="f5ea2-122">Only view two of the three message logs in the [Service Trace Viewer Tool (SvcTraceViewer.exe)](../../../../docs/framework/wcf/service-trace-viewer-tool-svctraceviewer-exe.md) at any time.</span></span>  
  
-   <span data-ttu-id="f5ea2-123">Si se deben ver todos los registros de tres de los [herramienta Service Trace Viewer (SvcTraceViewer.exe)](../../../../docs/framework/wcf/service-trace-viewer-tool-svctraceviewer-exe.md) al mismo tiempo, puede modificar el servicio de retransmisión mediante la creación de un nuevo <xref:System.ServiceModel.Channels.Message> instancia.</span><span class="sxs-lookup"><span data-stu-id="f5ea2-123">If you must view all three logs in the [Service Trace Viewer Tool (SvcTraceViewer.exe)](../../../../docs/framework/wcf/service-trace-viewer-tool-svctraceviewer-exe.md) at the same time, you can modify the relay service by creating a new <xref:System.ServiceModel.Channels.Message> instance.</span></span> <span data-ttu-id="f5ea2-124">Esta instancia debería ser una copia del cuerpo del mensaje entrante, más todos los encabezados salvo los encabezados `ActivityId` y `Action`.</span><span class="sxs-lookup"><span data-stu-id="f5ea2-124">This instance should be a copy of the body of the incoming message, plus all the headers except for the `ActivityId` and `Action` headers.</span></span> <span data-ttu-id="f5ea2-125">En el ejemplo siguiente de código se muestra cómo hacer esto:</span><span class="sxs-lookup"><span data-stu-id="f5ea2-125">The following example code demonstrates how to do this.</span></span>  
  
```  
Message outgoingMessage = Message.CreateMessage(incomingMessage.Version, incomingMessage.Headers.Action, incomingMessage.GetReaderAtBodyContents());  
  
for (int i = 0; i < incomingMessage.Headers.Count; i++)  
{  
   if (incomingMessage.Headers[i].Name.Equals("ActivityId", StringComparison.InvariantCultureIgnoreCase) ||  
incomingMessage.Headers[i].Name.Equals("Action", StringComparison.InvariantCultureIgnoreCase))  
   {  
      continue;  
    }  
    outgoingMessage.Headers.CopyHeaderFrom(incomingMessage, i);  
}  
```
