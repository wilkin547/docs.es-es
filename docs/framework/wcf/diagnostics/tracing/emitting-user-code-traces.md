ts.TraceTransfer(0, "transfer", traceID);
Trace.CorrelationManager.ActivityId = traceID; // Trace is static
ts.TraceEvent(TraceEventType.Start, 0, "Add request");
```

## <a name="propagating-the-activity-id-to-a-service"></a><span data-ttu-id="3ff77-124">Propagación del id. de la actividad a un servicio</span><span class="sxs-lookup"><span data-stu-id="3ff77-124">Propagating the Activity ID to A Service</span></span>

<span data-ttu-id="3ff77-125">Si establece el atributo `propagateActivity` en `true` para el origen de seguimiento de traza `System.ServiceModel` en los archivos de configuración del servicio y del cliente, el servicio que procesa la solicitud Add se produce en la misma actividad que la definida en el cliente.</span><span class="sxs-lookup"><span data-stu-id="3ff77-125">If you set the `propagateActivity` attribute to `true` for the `System.ServiceModel` trace source in both the client and service configuration files, the service processing for the Add request occurs in the same activity as the one defined in the client.</span></span> <span data-ttu-id="3ff77-126">Si el servicio define sus propias actividades y transferencias, las trazas del servicio no aparecen en la actividad propagada por el cliente.</span><span class="sxs-lookup"><span data-stu-id="3ff77-126">If the service defines its own activities and transfers, the service traces do not appear in the client-propagated activity.</span></span> <span data-ttu-id="3ff77-127">En su lugar, aparecen en una actividad que las trazas de la transferencia ponen en correlación con la actividad cuyo id. propaga el cliente.</span><span class="sxs-lookup"><span data-stu-id="3ff77-127">Instead, they appear in an activity correlated by transfer traces to the activity whose ID is propagated by the client.</span></span>

> [!NOTE]
> <span data-ttu-id="3ff77-128">Si el `propagateActivity` atributo se establece en `true` tanto en el cliente como en el servicio, WCF establece la actividad de ambiente en el ámbito de la operación del servicio.</span><span class="sxs-lookup"><span data-stu-id="3ff77-128">If the `propagateActivity` attribute is set to `true` on both the client and service, the ambient activity in the operation scope of the service is set by WCF.</span></span>

// equal to Guid.Empty), create a new one.

{
    Guid newGuid = Guid.NewGuid();
    Trace.CorrelationManager.ActivityId = newGuid;
}
// Emit your Start trace.

// Emit the processing traces for that request.

                            + n1 + ", " + n2);
// double result = n1 + n2;

// Emit the Stop trace and exit the method scope.

// return result;
```
