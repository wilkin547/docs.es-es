orientada a objetos, las relaciones entre dichos objetos son relativamente fáciles de navegar debido a que cada uno de ellos tiene un miembro que hace referencia a otro.</span><span class="sxs-lookup"><span data-stu-id="bf00a-107">In object-oriented programming, relationships between objects are relatively easy to navigate because each object has a member that references another object.</span></span> <span data-ttu-id="bf00a-108">Sin embargo, en tablas de bases de datos externas, navegar por relaciones no es tan sencillo.</span><span class="sxs-lookup"><span  <span data-ttu-id="bf00a-112">El marco de Language-Integrated Query (LINQ) proporciona dos operadores de combinación, <xref:System.Linq.Enumerable.Join%2A> y <xref:System.Linq.Enumerable.GroupJoin%2A> .</span><span class="sxs-lookup"><span data-stu-id="bf00a-112">The Language-Integrated Query (LINQ) framework provides two join operators, <xref:System.Linq.Enumerable.Join%2A> and <xref:System.Linq.Enumerable.GroupJoin%2A>.</span></span> <span data-ttu-id="bf00a-113">Estos operadores realizan *combinaciones de igualdad*: es decir, combinaciones que coinciden con dos orígenes de datos solo cuando sus claves son iguales.</span><span class="sxs-lookup"><span data-stu-id="bf00a-113">These operators perform *equi-joins*: that is, joins that match two data sources only when their keys are equal.</span></span> <span data-ttu-id="bf00a-114">(Por el contrario, Transact-SQL admite operadores de combinación distintos de `equals` , como el `less than` operador).</span><span class="sxs-lookup"><span data-stu-id="bf00a-114">(By contrast, Transact-SQL supports join operators other than `equals`, such as the `less than` operator.)</span></span>  
  
 <span data-ttu-id="bf00a-115">En cuanto a las bases de datos relacionales, <xref:System.Linq.Enumerable.Join%2A> implementa una combinación interna.</span><span class="sxs-lookup"><span data-stu-id="bf00a-115">In relational database terms, <xref:System.Linq.Enumerable.Join%2A> implements an inner join.</span></span> <span data-ttu-id="bf00a-116">Una combinación interna es un tipo de combinación en la que solamente se devuelven los objetos que tienen una correspondencia en el conjunto de datos opuesto.</span><span class="sxs-lookup"><span data-stu-id="bf00a-116">An inner join is a type of join in which only those objects that have a match in the opposite data set are returned.</span></span>  
  
 <span data-ttu-id="bf00a-117">Los <xref:System.Linq.Enumerable.GroupJoin%2A> operadores no tienen ningún equivalente directo en términos de bases de datos relacionales; implementan un superconjunto de combinaciones internas y combinaciones externas izquierdas.</span><span class="sxs-lookup"><span data-stu-id="bf00a-117">The <xref:System.Linq.Enumerable.GroupJoin%2A> operators have no direct equivalent in relational database terms; they implement a superset of inner joins and left outer joins.</span></span> <span data-ttu-id="bf00a-118">Una combinación externa izquierda es una combinación que devuelve cada elemento de la primera colección (izquierda), aunque éste no tenga elementos correlacionados en la segunda colección.</span><span class="sxs-lookup"><span data-stu-id="bf00a-118">A left outer join is a join that returns each element of the first (left) collection, even if it has no correlated elements in the second collection.</span></span>
