# <a name="signing-stored-procedures-in-sql-server"></a><span data-ttu-id="6877e-102">Firmar procedimientos almacenados en SQL Server</span><span class="sxs-lookup"><span data-stu-id="6877e-102">Signing Stored Procedures in SQL Server</span></span>
 <span data-ttu-id="6877e-103">Una firma digital es un resumen de datos cifrados con una clave privada del firmante.</span><span class="sxs-lookup"><span data-stu-id="6877e-103">A digital signature is a data digest encrypted with the private key of the signer.</span></span> <span data-ttu-id="6877e-104">La clave privada garantiza que la firma digital sea única para su portador o propietario.</span><span class="sxs-lookup"><span data-stu-id="6877e-104">The private key ensures that the digital signature is unique to its bearer or owner.</span></span> <span data-ttu-id="6877e-105">Puede firmar los ensamblados, funciones (excepto funciones con valores de tabla insertadas), desencadenadores y procedimientos almacenados.</span><span class="sxs-lookup"><span data-stu-id="6877e-105">You can sign stored procedures, functions (except for inline table-valued functions), triggers, and assemblies.</span></span>  
  
 <span data-ttu-id="6877e-106">Puede firmar un procedimiento almacenado con un certificado o una clave asimétrica.</span><span class="sxs-lookup"><span data-stu-id="6877e-106">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="6877e-107">Esto está diseñado para los casos en los que no se pueden heredar permisos mediante encadenamiento de propiedad o cuando la cadena de propiedad está rota, como en SQL dinámico.</span><span class="sxs-lookup"><span data-stu-id="6877e-107">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="6877e-108">A continuación, puede crear un usuario asignado al certificado, cual concede permisos de usuario en los objetos que el procedimiento almacenado necesita tener acceso a.</span><span class="sxs-lookup"><span data-stu-id="6877e-108">You can then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>  

 <span data-ttu-id="6877e-109">Puede también crear un inicio de sesión asignado para el mismo certificado y, a continuación, conceder los permisos de nivel de servidor necesarios para ese inicio de sesión o agregue el inicio de sesión a uno o varios de los roles fijos de servidor.</span><span class="sxs-lookup"><span data-stu-id="6877e-109">You can also create a login mapped to the same certificate, and then grant any necessary server-level permissions to that login, or add the login to one or more of the fixed server roles.</span></span> <span data-ttu-id="6877e-110">Esto está diseñado para evitar tener que habilitar la `TRUSTWORTHY` configuración para escenarios en los que se necesitan permisos de nivel superior de la base de datos.</span><span class="sxs-lookup"><span data-stu-id="6877e-110">This is designed to avoid enabling the `TRUSTWORTHY` database setting for scenarios in which higher level permissions are needed.</span></span>

## <a name="creating-certificates"></a><span data-ttu-id="6877e-114">Crear certificados</span><span class="sxs-lookup"><span data-stu-id="6877e-114">Creating Certificates</span></span>  
 <span data-ttu-id="6877e-115">Al firmar un procedimiento almacenado con un certificado o clave asimétrica, un resumen de datos que consta de hash cifrado de código del procedimiento almacenado, junto ejecución, el resumen de datos se descifra con la clave pública y se compara con el valor hash del procedimiento almacenado.</span><span class="sxs-lookup"><span data-stu-id="6877e-116">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="6877e-117">Cambiar el execute-como usuario invalida el valor hash para que ya no coincide con la firma digital.</span><span class="sxs-lookup"><span data-stu-id="6877e-117">Changing the execute-as user invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="6877e-118">Modificar el procedimiento almacenado quita la firma por completo, lo que impide que alguien que no tiene acceso a la clave privada pueda código o el de ejecución-como usuario.</span><span class="sxs-lookup"><span data-stu-id="6877e-119">In either case, you must re-sign the procedure each time you change the code or the execute-as user.</span></span>  
  
 <span data-ttu-id="6877e-120">Hay dos pasos implicados en la firma de un módulo:</span><span class="sxs-lookup"><span data-stu-id="6877e-120">There are two required steps involved in signing a module:</span></span>  
  
1.  <span data-ttu-id="6877e-121">Crear un certificado mediante la instrucción Transact-SQL `CREATE CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="6877e-121">Create a certificate using the Transact-SQL `CREATE CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="6877e-122">Esta instrucción tiene varias opciones para establecer una fecha de inicio y finalización, así como una contraseña.</span><span class="sxs-lookup"><span data-stu-id="6877e-122">This statement has several options for setting a start and end date and a password.</span></span> <span data-ttu-id="6877e-123">La fecha de expiración predeterminada es un año.</span><span class="sxs-lookup"><span data-stu-id="6877e-123">The default expiration date is one year.</span></span>  
  
1.  <span data-ttu-id="6877e-124">Firmar el procedimiento con el certificado utilizando la instrucción Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="6877e-124">Sign the procedure with the certificate using the Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` statement.</span></span>  

<span data-ttu-id="6877e-125">Una vez que se haya firmado el módulo, debe crearse con el fin de mantener los permisos adicionales que se deben asociados con el certificado de una o más entidades.</span><span class="sxs-lookup"><span data-stu-id="6877e-125">Once the module has been signed, one or more principals needs to be created in order to hold the additional permissions that should be associated with the certificate.</span></span>  

<span data-ttu-id="6877e-126">Si el módulo necesita permisos de nivel de base de datos adicionales:</span><span class="sxs-lookup"><span data-stu-id="6877e-126">If the module needs additional database-level permissions:</span></span>  
  
1.  <span data-ttu-id="6877e-127">Crear una base de datos asociada a un certificado utilizando la instrucción Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="6877e-127">Create a database user associated with that certificate using the Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="6877e-128">Este usuario existe en la base de datos únicamente y no está asociado a un inicio de sesión a menos que un inicio de sesión también creada a partir de ese mismo certificado.</span><span class="sxs-lookup"><span data-stu-id="6877e-128">This user exists in the database only and is not associated with a login unless a login has also been created from that same certificate.</span></span>  
  
1.  <span data-ttu-id="6877e-129">Conceder los permisos necesarios de nivel de base de datos de usuario del certificado.</span><span class="sxs-lookup"><span data-stu-id="6877e-129">Grant the certificate user the required database-level permissions.</span></span>  
  
<span data-ttu-id="6877e-130">Si el módulo necesita permisos de nivel de servidor adicionales:</span><span class="sxs-lookup"><span data-stu-id="6877e-130">If the module needs additional server-level permissions:</span></span>  
  
1.  <span data-ttu-id="6877e-131">Copie el certificado para el `master` base de datos.</span><span class="sxs-lookup"><span data-stu-id="6877e-131">Copy the certificate to the `master` database.</span></span>  
 
1.  <span data-ttu-id="6877e-132">Crear un inicio de sesión asociado con ese certificado mediante la instrucción Transact-SQL `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]` instrucción.</span><span class="sxs-lookup"><span data-stu-id="6877e-132">Create a login associated with that certificate using the Transact-SQL `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]` statement.</span></span>  
  
1.  <span data-ttu-id="6877e-133">Conceder el inicio de sesión de certificado los permisos de nivel de servidor necesarios.</span><span class="sxs-lookup"><span data-stu-id="6877e-133">Grant the certificate login the required server-level permissions.</span></span>  
  
> [!NOTE]  
>  <span data-ttu-id="6877e-134">Un certificado no puede conceder permisos a un usuario que tiene permisos revocados con la instrucción DENY.</span><span class="sxs-lookup"><span data-stu-id="6877e-134">A certificate cannot grant permissions to a user that has had permissions revoked using the DENY statement.</span></span> <span data-ttu-id="6877e-135">DENY siempre tiene prioridad sobre GRANT, lo que evita que el llamador herede permisos concedidos al usuario del certificado.</span><span class="sxs-lookup"><span data-stu-id="6877e-135">DENY always takes precedence over GRANT, preventing the caller from inheriting permissions granted to the certificate user.</span></span>
