## <a name="common-threats"></a><span data-ttu-id="01dec-109">Amenazas comunes</span><span class="sxs-lookup"><span data-stu-id="01dec-109">Common Threats</span></span>  
 <span data-ttu-id="01dec-110">Los desarrolladores han de saber cuáles son las amenazas a la seguridad, las herramientas con las que cuentan para enfrentarse a ellas y cómo evitar las vulnerabilidades de seguridad provocadas por los propios usuarios.</span><span class="sxs-lookup"><span data-stu-id="01dec-110">Developers need to understand security threats, the tools provided to counter them, and how to avoid self-inflicted security holes.</span></span> <span data-ttu-id="01dec-111">Se puede pensar en la seguridad como en una cadena, en la que si se rompe un vínculo, se pone en peligro la fortaleza de toda ella.</span><span class="sxs-lookup"><span data-stu-id="01dec-111">Security can best be thought of as a chain, where a break in any one link compromises the strength of the whole.</span></span> <span data-ttu-id="01dec-112">En la siguiente lista se incluyen algunas de las amenazas a la seguridad más comunes, que se comentan en mayor detalle en los temas de este apartado.</span><span class="sxs-lookup"><span data-stu-id="01dec-112">The following list includes some common security threats that are discussed in more detail in the topics in this section.</span></span>  
  
### <a name="sql-injection"></a><span data-ttu-id="01dec-113">inyección de código SQL</span><span class="sxs-lookup"><span data-stu-id="01dec-113">SQL Injection</span></span>  
 <span data-ttu-id="01dec-114">La inyección de SQL es el proceso por el cual un usuario malintencionado escribe instrucciones de Transact-SQL en lugar de entradas válidas.</span><span class="sxs-lookup"><span data-stu-id="01dec-114">SQL Injection is the process by which a malicious user enters Transact-SQL statements instead of valid input.</span></span> <span data-ttu-id="01dec-115">Si la entrada se pasa directamente al servidor sin haber sido validada y la aplicación ejecuta el código inyectado por error, el ataque podría dañar o destruir datos.</span><span class="sxs-lookup"><span data-stu-id="01dec-115">If the input is passed directly to the server without being validated and if the application inadvertently executes the injected code, then the attack has the potential to damage or destroy data.</span></span> <span data-ttu-id="01dec-116">Para frustrar los ataques por inyección de SQL Server, puede utilizar procedimientos almacenados y comandos parametrizados, evitar el SQL dinámico y restringir los permisos de todos los usuarios.</span><span class="sxs-lookup"><span data-stu-id="01dec-116">You can thwart SQL Server injection attacks by using stored procedures and parameterized commands, avoiding dynamic SQL, and restricting permissions on all users.</span></span>  
  
### <a name="elevation-of-privilege"></a><span data-ttu-id="01dec-117">Elevación de privilegios</span><span class="sxs-lookup"><span data-stu-id="01dec-117">Elevation of Privilege</span></span>  
 <span data-ttu-id="01dec-118">Los ataques por elevación de privilegios se producen cuando un usuario es capaz de asumir los privilegios de una cuenta de confianza, como un propietario o un administrador.</span><span class="sxs-lookup"><span data-stu-id="01dec-118">Elevation of privilege attacks occur when a user is able to assume the privileges of a trusted account, such as an owner or administrator.</span></span> <span data-ttu-id="01dec-119">Trabaje siempre en cuentas de usuario con los privilegios mínimos y asigne sólo los permisos necesarios.</span><span class="sxs-lookup"><span data-stu-id="01dec-119">Always run under least-privileged user accounts and assign only needed permissions.</span></span> <span data-ttu-id="01dec-120">Evite utilizar cuentas administrativas o de propietario para ejecutar código.</span><span class="sxs-lookup"><span data-stu-id="01dec-120">Avoid using administrative or owner accounts for executing code.</span></span> <span data-ttu-id="01dec-121">De esta forma se limita el daño que se podría sufrir en caso de que un ataque tenga éxito.</span><span class="sxs-lookup"><span data-stu-id="01dec-121">This limits the amount of damage that can occur if an attack succeeds.</span></span> <span data-ttu-id="01dec-122">Cuando realice tareas que requieran permisos adicionales, utilice la firma de procedimientos o la suplantación únicamente mientras dure la tarea. </span><span class="sxs-lookup"><span data-stu-id="01dec-122">When performing tasks that require additional permissions, use procedure signing or impersonation only for the duration of the task.</span></span> <span data-ttu-id="01dec-123">Puede firmar procedimientos almacenados con certificados o usar la suplantación para asignar permisos temporalmente.</span><span class="sxs-lookup"><span data-stu-id="01dec-123">You can sign stored procedures with certificates or use impersonation to temporarily assign permissions.</span></span>  
  
### <a name="probing-and-intelligent-observation"></a><span data-ttu-id="01dec-124">Sondeos y observación inteligente</span><span class="sxs-lookup"><span data-stu-id="01dec-124">Probing and Intelligent Observation</span></span>

### <a name="authentication"></a><span data-ttu-id="01dec-127">Autenticación</span><span class="sxs-lookup"><span data-stu-id="01dec-127">Authentication</span></span>  
 <span data-ttu-id="01dec-128">Cuando se utilizan inicios de sesión de SQL Server, se pueden producir ataques por inyección a la cadena de conexión si durante la ejecución se genera una cadena de conexión basada en los datos introducidos por el usuario. </span><span class="sxs-lookup"><span data-stu-id="01dec-128">A connection string injection attack can occur when using SQL Server logins if a connection string based on user input is constructed at run time.</span></span> <span data-ttu-id="01dec-129">Si en la cadena de conexión no se comprueba la validez de los pares de palabras clave, un agresor puede insertar caracteres de más y así podría tener acceso a datos confidenciales o a otros recursos del servidor.</span><span class="sxs-lookup"><span data-stu-id="01dec-129">If the connection string is not checked for valid keyword pairs, an attacker can insert extra characters, potentially accessing sensitive data or other resources on the server.</span></span> <span data-ttu-id="01dec-130">Utilice la autenticación de Windows siempre que sea posible.</span><span class="sxs-lookup"><span data-stu-id="01dec-130">Use Windows authentication wherever possible.</span></span> <span data-ttu-id="01dec-131">Si tiene que utilizar inicios de sesión de SQL Server, use el <xref:System.Data.SqlClient.SqlConnectionStringBuilder> para crear y validar cadenas de conexión durante la ejecución.</span><span class="sxs-lookup"><span data-stu-id="01dec-131">If you must use SQL Server logins, use the <xref:System.Data.SqlClient.SqlConnectionStringBuilder> to create and validate connection strings at run time.</span></span>  
  
### <a name="passwords"></a><span data-ttu-id="01dec-132">Contraseñas</span><span class="sxs-lookup"><span data-stu-id="01dec-132">Passwords</span></span>  
 <span data-ttu-id="01dec-133">Muchos ataques tienen éxito porque un intruso es capaz de obtener o adivinar la contraseña de un usuario con muchos privilegios.</span><span class="sxs-lookup"><span data-stu-id="01dec-133">Many attacks succeed because an intruder was able to obtain or guess a password for a privileged user.</span></span> <span data-ttu-id="01dec-134">Las contraseñas constituyen su primera línea de defensa contra los intrusos, así que establecer contraseñas seguras es esencial para la seguridad del sistema.</span><span class="sxs-lookup"><span data-stu-id="01dec-134">Passwords are your first line of defense against intruders, so setting strong passwords is essential to the security of your system.</span></span> <span data-ttu-id="01dec-135">Cree y aplique directivas de contraseñas para la autenticación en modo mixto.</span><span class="sxs-lookup"><span data-stu-id="01dec-135">Create and enforce password policies for mixed mode authentication.</span></span>
