# <a name="compiler-warning-level-1-cs0420"></a>Advertencia del compilador (nivel 1) CS0420

'identifier': una referencia a un campo volátil no se tratará como tal  
  
 Normalmente, no se debe pasar un campo volatile mediante un parámetro `ref` u **out**, puesto que no se tratará como volátil dentro del ámbito de la función. Existen excepciones a esta norma, como cuando se llama llamar a una API entrelazada. Como con cualquier advertencia, puede usar [#pragma warning](../preprocessor-directives.md#pragma-warning) para deshabilitar esta advertencia en los infrecuentes casos en que se use un campo volatile de manera intencionada como parámetro de referencia.  
  
 En el ejemplo siguiente se genera la advertencia CS0420:  
  
```csharp  
// CS0420.cs  
// compile with: /W:1  
using System;  
  
class TestClass  
{  
   private volatile int i;  
  
   public void TestVolatile(ref int ii)  
   {  
   }  
  
   public static void Main()  
   {  
      TestClass x = new TestClass();  
      x.TestVolatile(ref x.i);   // CS0420
   }  
}  
```
