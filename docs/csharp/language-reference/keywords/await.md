<span data-ttu-id="910a2-103">El operador `await` se aplica a una tarea de un método asincrónico para insertar un punto de suspensión en la ejecución del método hasta que la tarea en espera se complete.</span><span class="sxs-lookup"><span data-stu-id="910a2-103">The `await` operator is applied to a task in an asynchronous method to insert a suspension point in the execution of the method until the awaited task completes.</span></span> <span data-ttu-id="910a2-104">La tarea representa el trabajo en curso.</span><span class="sxs-lookup"><span data-stu-id="910a2-104">The task represents ongoing work.</span></span>  
  
<span data-ttu-id="910a2-105">`await` solo puede usarse en un método asincrónico que se ha modificado mediante la palabra clave [async](../../../csharp/language-reference/keywords/async.md).</span><span class="sxs-lookup"><span data-stu-id="910a2-105">`await` can only be used in an asynchronous method modified by the [async](../../../csharp/language-reference/keywords/async.md) keyword.</span></span> <span data-ttu-id="910a2-106">Este tipo de método, que se define mediante el modificador `async` y que generalmente contiene una o más expresiones `await`, se denomina *método asincrónico*.</span><span class="sxs-lookup"><span data-stu-id="910a2-106">Such a method, defined by using the `async` modifier and usually containing one or more `await` expressions, is referred to as an *async method*.</span></span>

<span data-ttu-id="910a2-109">La tarea a la que se aplica el operador `await` se devuelve normalmente mediante una llamada a un método que implementa el [patrón asincrónico basado en tareas](../../../standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md).</span><span class="sxs-lookup"><span data-stu-id="910a2-109">The task to which the `await` operator is applied typically is returned by a call to a method that implements the [Task-Based Asynchronous Pattern](../../../standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md).</span></span> <span data-ttu-id="910a2-110">Incluyen métodos que devuelven objetos <xref:System.Threading.Tasks.Task>, <xref:System.Threading.Tasks.Task%601> y `System.Threading.Tasks.ValueType<TResult>`.</span><span class="sxs-lookup"><span data-stu-id="910a2-110">They include methods that return <xref:System.Threading.Tasks.Task>, <xref:System.Threading.Tasks.Task%601>, and `System.Threading.Tasks.ValueType<TResult>` objects.</span></span>  

  
 <span data-ttu-id="910a2-111">En el siguiente ejemplo, el método <xref:System.Net.Http.HttpClient.GetByteArrayAsync%2A?displayProperty=nameWithType> devuelve un `Task<byte[]>`.</span><span class="sxs-lookup"><span data-stu-id="910a2-111">In the following example, the <xref:System.Net.Http.HttpClient.GetByteArrayAsync%2A?displayProperty=nameWithType> method returns a `Task<byte[]>`.</span></span> <span data-ttu-id="910a2-112">La tarea garantiza que la matriz de bytes real se generará cuando finalice la tarea.</span><span class="sxs-lookup"><span data-stu-id="910a2-112">The task is a promise to produce the actual byte array when the task is complete.</span></span> <span data-ttu-id="910a2-113">El operador `await` suspende la ejecución hasta que el trabajo del método <xref:System.Net.Http.HttpClient.GetByteArrayAsync%2A> se completa.</span><span class="sxs-lookup"><span data-stu-id="910a2-113">The `await` operator suspends execution until the work of the <xref:System.Net.Http.HttpClient.GetByteArrayAsync%2A> method is complete.</span></span> <span data-ttu-id="910a2-114">Mientras tanto, el control vuelve al llamador de `GetPageSizeAsync`.</span><span class="sxs-lookup"><span data-stu-id="910a2-114">In the meantime, control is returned to the caller of `GetPageSizeAsync`.</span></span> <span data-ttu-id="910a2-115">Cuando la tarea finaliza la ejecución, la expresión `await` se evalúa como una matriz de bytes.</span><span class="sxs-lookup"><span data-stu-id="910a2-115">When the task finishes execution, the `await` expression evaluates to a byte array.</span></span>

<span data-ttu-id="910a2-122">Una expresión `await` no bloquea el subproceso en el que se ejecuta.</span><span class="sxs-lookup"><span data-stu-id="910a2-122">An `await` expression does not block the thread on which it is executing.</span></span> <span data-ttu-id="910a2-123">En su lugar, hace que el compilador suscriba el resto del método asincrónico como una continuación de la tarea esperada.</span><span class="sxs-lookup"><span data-stu-id="910a2-123">Instead, it causes the compiler to sign up the rest of the async method as a continuation on the awaited task.</span></span> <span data-ttu-id="910a2-124">A continuación, el control vuelve al llamador del método asincrónico.</span><span class="sxs-lookup"><span data-stu-id="910a2-124">Control then returns to the caller of the async method.</span></span> <span data-ttu-id="910a2-125">Cuando la tarea se completa, invoca su continuación y la ejecución del método asincrónico se reanuda donde se quedó.</span><span class="sxs-lookup"><span data-stu-id="910a2-125">When the task completes, it invokes its continuation, and execution of the async method resumes where it left off.</span></span>  
  
<span data-ttu-id="910a2-126">Una expresión `await` solo puede aparecer en el cuerpo de su método envolvente, en una expresión lambda o en un método anónimo que debe marcarse con un modificador `async`.</span><span class="sxs-lookup"><span data-stu-id="910a2-126">An `await` expression can occur only in the body of its enclosing method, lambda expression, or anonymous method, which must be marked with an `async` modifier.</span></span> <span data-ttu-id="910a2-127">El término *await* solo sirve como palabra clave en ese contexto.</span><span class="sxs-lookup"><span data-stu-id="910a2-127">The term *await* serves as a keyword only in that context.</span></span> <span data-ttu-id="910a2-128">En cualquier otra parte, se interpretará como identificador.</span><span class="sxs-lookup"><span data-stu-id="910a2-128">Elsewhere, it is interpreted as an identifier.</span></span> <span data-ttu-id="910a2-129">Dentro del método, expresión lambda o método anónimo, una expresión `await` no se puede producir en el cuerpo de una función sincrónica, en una expresión de consulta, en el bloque de una [instrucción lock](../../../csharp/language-reference/keywords/lock-statement.md), ni en un contexto [unsafe](../../../csharp/language-reference/keywords/unsafe.md).</span><span class="sxs-lookup"><span data-stu-id="910a2-129">Within the method, lambda expression, or anonymous method, an `await` expression cannot occur in the body of a synchronous function, in a query expression, in the block of a [lock statement](../../../csharp/language-reference/keywords/lock-statement.md), or in an [unsafe](../../../csharp/language-reference/keywords/unsafe.md) context.</span></span>  
  
## <a name="exceptions"></a><span data-ttu-id="910a2-130">Excepciones</span><span class="sxs-lookup"><span data-stu-id="910a2-130">Exceptions</span></span>  
<span data-ttu-id="910a2-131">La mayoría de los métodos asincrónicos devuelven un objeto <xref:System.Threading.Tasks.Task> o <xref:System.Threading.Tasks.Task%601>.</span><span class="sxs-lookup"><span data-stu-id="910a2-131">Most async methods return a <xref:System.Threading.Tasks.Task> or <xref:System.Threading.Tasks.Task%601>.</span></span> <span data-ttu-id="910a2-132">Las propiedades de la tarea devuelta llevan información acerca de su estado e historial, por ejemplo, si la tarea se ha completado, si el método asincrónico produjo una excepción o si se ha cancelado y cuál es el resultado final.</span><span class="sxs-lookup"><span data-stu-id="910a2-132">The properties of the returned task carry information about its status and history, such as whether the task is complete, whether the async method caused an exception or was canceled, and what the final result is.</span></span> <span data-ttu-id="910a2-133">El operador `await` tiene acceso a esas propiedades llamando a métodos en el objeto devuelto mediante el método `GetAwaiter`.</span><span class="sxs-lookup"><span data-stu-id="910a2-133">The `await` operator accesses those properties by calling methods on the object returned by the `GetAwaiter` method.</span></span>  
  
<span data-ttu-id="910a2-134">Si espera un método asincrónico que devuelve una tarea y causa una excepción, el operador `await` volverá a generar la excepción.</span><span class="sxs-lookup"><span data-stu-id="910a2-134">If you await a task-returning async method that causes an exception, the `await` operator rethrows the exception.</span></span>  
  
<span data-ttu-id="910a2-135">Si espera un método asincrónico que devuelve una tarea y se cancela, el operador `await` volverá a generar <xref:System.OperationCanceledException>.</span><span class="sxs-lookup"><span data-stu-id="910a2-135">If you await a task-returning async method that's canceled, the `await` operator rethrows an <xref:System.OperationCanceledException>.</span></span>  
  
<span data-ttu-id="910a2-136">Una única tarea en estado de error puede reflejar varias excepciones.</span><span class="sxs-lookup"><span data-stu-id="910a2-136">A single task that is in a faulted state can reflect multiple exceptions.</span></span> <span data-ttu-id="910a2-137">Por ejemplo, la tarea podría ser el resultado de una llamada a <xref:System.Threading.Tasks.Task.WhenAll%2A?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="910a2-137">For example, the task might be the result of a call to <xref:System.Threading.Tasks.Task.WhenAll%2A?displayProperty=nameWithType>.</span></span> <span data-ttu-id="910a2-138">Cuando espera dicha tarea, la operación await vuelve a generar únicamente una de las excepciones.</span><span class="sxs-lookup"><span data-stu-id="910a2-138">When you await such a task, the await operation rethrows only one of the exceptions.</span></span> <span data-ttu-id="910a2-139">Sin embargo, no se puede predecir cuál de las excepciones se vuelve a generar.</span><span class="sxs-lookup"><span data-stu-id="910a2-139">However, you can't predict which of the exceptions is rethrown.</span></span>  
  
<span data-ttu-id="910a2-140">Para obtener ejemplos de control de errores en métodos asincrónicos, vea [try-catch](../../../csharp/language-reference/keywords/try-catch.md).</span><span class="sxs-lookup"><span data-stu-id="910a2-140">For examples of error handling in async methods, see [try-catch](../../../csharp/language-reference/keywords/try-catch.md).</span></span>
