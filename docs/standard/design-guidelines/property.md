# <a name="property-design"></a><span data-ttu-id="6b24d-103">Diseño de propiedades</span><span class="sxs-lookup"><span data-stu-id="6b24d-103">Property Design</span></span>

<span data-ttu-id="6b24d-104">Aunque las propiedades son técnicamente muy similares a los métodos, son bastante diferentes en cuanto a sus escenarios de uso.</span><span class="sxs-lookup"><span data-stu-id="6b24d-104">Although properties are technically very similar to methods, they are quite different in terms of their usage scenarios.</span></span> <span data-ttu-id="6b24d-105">Deberían verse como campos inteligentes.</span><span class="sxs-lookup"><span data-stu-id="6b24d-105">They should be seen as smart fields.</span></span> <span data-ttu-id="6b24d-106">Tienen la sintaxis de llamada de los campos y la flexibilidad de los métodos.</span><span class="sxs-lookup"><span data-stu-id="6b24d-106">They have the calling syntax of fields, and the flexibility of methods.</span></span>

 <span data-ttu-id="6b24d-107">✔️ CREE solo propiedades get si no debe permitirse que el llamador cambie el valor de la propiedad.</span><span class="sxs-lookup"><span data-stu-id="6b24d-107">✔️ DO create get-only properties if the caller should not be able to change the value of the property.</span></span>

 <span data-ttu-id="6b24d-108">Tenga en cuenta que si el tipo de la propiedad es un tipo de referencia mutable, el valor de propiedad se puede cambiar incluso si la propiedad solo es get.</span><span class="sxs-lookup"><span data-stu-id="6b24d-108">Keep in mind that if the type of the property is a mutable reference type, the property value can be changed even if the property is get-only.</span></span>

 <span data-ttu-id="6b24d-109">❌ NO proporcione solo propiedades set o propiedades en las que el establecedor tenga una accesibilidad más amplia que el captador.</span><span class="sxs-lookup"><span data-stu-id="6b24d-109">❌ DO NOT provide set-only properties or properties with the setter having broader accessibility than the getter.</span></span>

 <span data-ttu-id="6b24d-110">Por ejemplo, no use propiedades con un establecedor público y un captador protegido.</span><span class="sxs-lookup"><span data-stu-id="6b24d-110">For example, do not use properties with a public setter and a protected getter.</span></span>

 <span data-ttu-id="6b24d-111">Si no se puede proporcionar el captador de propiedades, implemente la funcionalidad como método en su lugar.</span><span class="sxs-lookup"><span data-stu-id="6b24d-111">If the property getter cannot be provided, implement the functionality as a method instead.</span></span> <span data-ttu-id="6b24d-112">Considere la posibilidad de empezar el nombre del método por `Set` y siga con el nombre que le habría dado a la propiedad.</span><span class="sxs-lookup"><span data-stu-id="6b24d-112">Consider starting the method name with `Set` and follow with what you would have named the property.</span></span> <span data-ttu-id="6b24d-113">Por ejemplo, <xref:System.AppDomain> tiene un método llamado `SetCachePath` en lugar de tener solo una propiedad set denominada `CachePath`.</span><span class="sxs-lookup"><span data-stu-id="6b24d-113">For example, <xref:System.AppDomain> has a method called `SetCachePath` instead of having a set-only property called `CachePath`.</span></span>

 <span data-ttu-id="6b24d-114">✔️ PROPORCIONE valores predeterminados razonables de todas las propiedades, asegurándose de que los valores predeterminados no dan lugar a una vulnerabilidad de seguridad ni a un código tremendamente ineficaz.</span><span class="sxs-lookup"><span data-stu-id="6b24d-114">✔️ DO provide sensible default values for all properties, ensuring that the defaults do not result in a security hole or terribly inefficient code.</span></span>

 <span data-ttu-id="6b24d-115">✔️ PERMITA que las propiedades se establezcan en cualquier orden, incluso si el resultado es un estado no válido temporal del objeto.</span><span class="sxs-lookup"><span data-stu-id="6b24d-115">✔️ DO allow properties to be set in any order even if this results in a temporary invalid state of the object.</span></span>

 <span data-ttu-id="6b24d-116">Es común que dos o más propiedades estén interrelacionadas con un punto en el que algunos valores de una propiedad podrían no ser válidos dados los valores de otras propiedades en el mismo objeto.</span><span class="sxs-lookup"><span data-stu-id="6b24d-116">It is common for two or more properties to be interrelated to a point where some values of one property might be invalid given the values of other properties on the same object.</span></span> <span data-ttu-id="6b24d-117">En tales casos, las excepciones resultantes del estado no válido se deben posponer hasta que el objeto utilice realmente las propiedades interrelacionadas de forma conjunta.</span><span class="sxs-lookup"><span data-stu-id="6b24d-117">In such cases, exceptions resulting from the invalid state should be postponed until the interrelated properties are actually used together by the object.</span></span>

 <span data-ttu-id="6b24d-118">✔️ CONSERVE el valor anterior si un establecedor de propiedades inicia una excepción.</span><span class="sxs-lookup"><span data-stu-id="6b24d-118">✔️ DO preserve the previous value if a property setter throws an exception.</span></span>

 <span data-ttu-id="6b24d-119">❌ EVITE iniciar excepciones desde captadores de propiedades.</span><span class="sxs-lookup"><span data-stu-id="6b24d-119">❌ AVOID throwing exceptions from property getters.</span></span>

 <span data-ttu-id="6b24d-120">Los captadores de propiedades deben ser operaciones simples y no deben tener ninguna condición previa.</span><span class="sxs-lookup"><span data-stu-id="6b24d-120">Property getters should be simple operations and should not have any preconditions.</span></span> <span data-ttu-id="6b24d-121">Si un captador puede iniciar una excepción, es probable que haya que rediseñarlo para que sea un método.</span><span class="sxs-lookup"><span data-stu-id="6b24d-121">If a getter can throw an exception, it should probably be redesigned to be a method.</span></span> <span data-ttu-id="6b24d-122">Tenga en cuenta que esta regla no se aplica a los indizadores, donde esperamos excepciones como resultado de la validación de los argumentos.</span><span class="sxs-lookup"><span data-stu-id="6b24d-122">Notice that this rule does not apply to indexers, where we do expect exceptions as a result of validating the arguments.</span></span>

### <a name="indexed-property-design"></a><span data-ttu-id="6b24d-123">Diseño de propiedades indizadas</span><span class="sxs-lookup"><span data-stu-id="6b24d-123">Indexed Property Design</span></span>

 <span data-ttu-id="6b24d-124">Una propiedad indizada es una propiedad especial que puede tener parámetros y a la que se puede llamar con una sintaxis especial similar a la indexación de matrices.</span><span class="sxs-lookup"><span data-stu-id="6b24d-124">An indexed property is a special property that can have parameters and can be called with special syntax similar to array indexing.</span></span>

 <span data-ttu-id="6b24d-125">Las propiedades indexadas se conocen normalmente como indizadores.</span><span class="sxs-lookup"><span data-stu-id="6b24d-125">Indexed properties are commonly referred to as indexers.</span></span> <span data-ttu-id="6b24d-126">Los indizadores deben usarse solo en las API que proporcionan acceso a los elementos de una colección lógica.</span><span class="sxs-lookup"><span data-stu-id="6b24d-126">Indexers should be used only in APIs that provide access to items in a logical collection.</span></span> <span data-ttu-id="6b24d-127">Por ejemplo, una cadena es una colección de caracteres y el indizador de <xref:System.String?displayProperty=nameWithType> se agregó para tener acceso a sus caracteres.</span><span class="sxs-lookup"><span data-stu-id="6b24d-127">For example, a string is a collection of characters, and the indexer on <xref:System.String?displayProperty=nameWithType> was added to access its characters.</span></span>

 <span data-ttu-id="6b24d-128">✔️ CONSIDERE la posibilidad de usar indizadores para proporcionar acceso a los datos almacenados en una matriz interna.</span><span class="sxs-lookup"><span data-stu-id="6b24d-128">✔️ CONSIDER using indexers to provide access to data stored in an internal array.</span></span>

 <span data-ttu-id="6b24d-129">✔️ CONSIDERE la posibilidad de proporcionar indizadores en tipos que representan colecciones de elementos.</span><span class="sxs-lookup"><span data-stu-id="6b24d-129">✔️ CONSIDER providing indexers on types representing collections of items.</span></span>

 <span data-ttu-id="6b24d-130">❌ EVITE usar propiedades indexadas con más de un parámetro.</span><span class="sxs-lookup"><span data-stu-id="6b24d-130">❌ AVOID using indexed properties with more than one parameter.</span></span>

 <span data-ttu-id="6b24d-131">Si el diseño requiere varios parámetros, reconsidere si la propiedad realmente representa un descriptor de acceso a una colección lógica.</span><span class="sxs-lookup"><span data-stu-id="6b24d-131">If the design requires multiple parameters, reconsider whether the property really represents an accessor to a logical collection.</span></span> <span data-ttu-id="6b24d-132">Si no es así, use los métodos en su lugar.</span><span class="sxs-lookup"><span data-stu-id="6b24d-132">If it does not, use methods instead.</span></span> <span data-ttu-id="6b24d-133">Considere la posibilidad de empezar el nombre del método por `Get` o `Set`.</span><span class="sxs-lookup"><span data-stu-id="6b24d-133">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="6b24d-134">❌ EVITE los indizadores con tipos de parámetros distintos de <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType> o una enumeración.</span><span class="sxs-lookup"><span data-stu-id="6b24d-134">❌ AVOID indexers with parameter types other than <xref:System.Int32?displayProperty=nameWithType>, <xref:System.Int64?displayProperty=nameWithType>, <xref:System.String?displayProperty=nameWithType>, <xref:System.Object?displayProperty=nameWithType>, or an enum.</span></span>

 <span data-ttu-id="6b24d-135">Si el diseño requiere otros tipos de parámetros, vuelva a evaluar con firmeza si la API representa realmente un descriptor de acceso a una colección lógica.</span><span class="sxs-lookup"><span data-stu-id="6b24d-135">If the design requires other types of parameters, strongly reevaluate whether the API really represents an accessor to a logical collection.</span></span> <span data-ttu-id="6b24d-136">En caso contrario, use un método.</span><span class="sxs-lookup"><span data-stu-id="6b24d-136">If it does not, use a method.</span></span> <span data-ttu-id="6b24d-137">Considere la posibilidad de empezar el nombre del método por `Get` o `Set`.</span><span class="sxs-lookup"><span data-stu-id="6b24d-137">Consider starting the method name with `Get` or `Set`.</span></span>

 <span data-ttu-id="6b24d-138">✔️ USE el nombre `Item` de las propiedades indexadas, a menos que haya un nombre claramente mejor (p. ej., consulte la propiedad <xref:System.String.Chars%2A> en `System.String`).</span><span class="sxs-lookup"><span data-stu-id="6b24d-138">✔️ DO use the name `Item` for indexed properties unless there is an obviously better name (e.g., see the <xref:System.String.Chars%2A> property on `System.String`).</span></span>

 <span data-ttu-id="6b24d-139">En C#, a los indizadores se les llama, de forma predeterminada, Elemento.</span><span class="sxs-lookup"><span data-stu-id="6b24d-139">In C#, indexers are by default named Item.</span></span> <span data-ttu-id="6b24d-140"><xref:System.Runtime.CompilerServices.IndexerNameAttribute> se puede usar para personalizar este nombre.</span><span class="sxs-lookup"><span data-stu-id="6b24d-140">The <xref:System.Runtime.CompilerServices.IndexerNameAttribute> can be used to customize this name.</span></span>

 <span data-ttu-id="6b24d-141">❌ NO proporcione un indizador ni métodos que sean semánticamente equivalentes.</span><span class="sxs-lookup"><span data-stu-id="6b24d-141">❌ DO NOT provide both an indexer and methods that are semantically equivalent.</span></span>

 <span data-ttu-id="6b24d-142">❌ NO proporcione más de una familia de indizadores sobrecargados en un tipo.</span><span class="sxs-lookup"><span data-stu-id="6b24d-142">❌ DO NOT provide more than one family of overloaded indexers in one type.</span></span>

 <span data-ttu-id="6b24d-143">El compilador de C# aplica esto.</span><span class="sxs-lookup"><span data-stu-id="6b24d-143">This is enforced by the C# compiler.</span></span>

 <span data-ttu-id="6b24d-144">❌ NO use propiedades indexadas no predeterminadas.</span><span class="sxs-lookup"><span data-stu-id="6b24d-144">❌ DO NOT use nondefault indexed properties.</span></span>

 <span data-ttu-id="6b24d-145">El compilador de C# aplica esto.</span><span class="sxs-lookup"><span data-stu-id="6b24d-145">This is enforced by the C# compiler.</span></span>

### <a name="property-change-notification-events"></a><span data-ttu-id="6b24d-146">Eventos de notificación de cambio de propiedad</span><span class="sxs-lookup"><span data-stu-id="6b24d-146">Property Change Notification Events</span></span>

 <span data-ttu-id="6b24d-147">A veces resulta útil proporcionar un evento que notifique al usuario los cambios en un valor de propiedad.</span><span class="sxs-lookup"><span data-stu-id="6b24d-147">Sometimes it is useful to provide an event notifying the user of changes in a property value.</span></span> <span data-ttu-id="6b24d-148">Por ejemplo, `System.Windows.Forms.Control` genera un evento `TextChanged` después de que el valor de su propiedad `Text` haya cambiado.</span><span class="sxs-lookup"><span data-stu-id="6b24d-148">For example, `System.Windows.Forms.Control` raises a `TextChanged` event after the value of its `Text` property has changed.</span></span>

 <span data-ttu-id="6b24d-149">✔️ CONSIDERE la posibilidad de generar eventos de notificación de cambios cuando se modifiquen los valores de propiedad en las API de alto nivel (normalmente los componentes del diseñador).</span><span class="sxs-lookup"><span data-stu-id="6b24d-149">✔️ CONSIDER raising change notification events when property values in high-level APIs (usually designer components) are modified.</span></span>

 <span data-ttu-id="6b24d-150">Si hay un buen escenario para que un usuario sepa cuándo cambia una propiedad de un objeto, el objeto debe generar un evento de notificación de cambio para la propiedad.</span><span class="sxs-lookup"><span data-stu-id="6b24d-150">If there is a good scenario for a user to know when a property of an object is changing, the object should raise a change notification event for the property.</span></span>

 <span data-ttu-id="6b24d-151">Sin embargo, es improbable que merezca la pena la sobrecarga de generar tales eventos para las API de bajo nivel, como los tipos base o las colecciones.</span><span class="sxs-lookup"><span data-stu-id="6b24d-151">However, it is unlikely to be worth the overhead to raise such events for low-level APIs such as base types or collections.</span></span> <span data-ttu-id="6b24d-152">Por ejemplo, <xref:System.Collections.Generic.List%601> no generaría estos eventos cuando se agrega un nuevo elemento a la lista y cambia la propiedad `Count`.</span><span class="sxs-lookup"><span data-stu-id="6b24d-152">For example, <xref:System.Collections.Generic.List%601> would not raise such events when a new item is added to the list and the `Count` property changes.</span></span>

 <span data-ttu-id="6b24d-153">✔️ CONSIDERE la posibilidad de generar eventos de notificación de cambio cuando el valor de una propiedad cambie a través de fuerzas externas.</span><span class="sxs-lookup"><span data-stu-id="6b24d-153">✔️ CONSIDER raising change notification events when the value of a property changes via external forces.</span></span>

 <span data-ttu-id="6b24d-154">Si un valor de propiedad cambia a través de una fuerza externa (de forma distinta que llamando a los métodos en el objeto), los eventos de generación indican al desarrollador que el valor cambia y ha cambiado.</span><span class="sxs-lookup"><span data-stu-id="6b24d-154">If a property value changes via some external force (in a way other than by calling methods on the object), raise events indicate to the developer that the value is changing and has changed.</span></span> <span data-ttu-id="6b24d-155">Un buen ejemplo es la propiedad `Text` de un control de cuadro de texto.</span><span class="sxs-lookup"><span data-stu-id="6b24d-155">A good example is the `Text` property of a text box control.</span></span> <span data-ttu-id="6b24d-156">Cuando el usuario escribe texto en un `TextBox`, el valor de propiedad cambia automáticamente.</span><span class="sxs-lookup"><span data-stu-id="6b24d-156">When the user types text in a `TextBox`, the property value automatically changes.</span></span>
