# <a name="general-naming-conventions"></a><span data-ttu-id="138cb-103">Convenciones generales de nomenclatura</span><span class="sxs-lookup"><span data-stu-id="138cb-103">General Naming Conventions</span></span>

<span data-ttu-id="138cb-104">En esta sección se describen las convenciones de nomenclatura generales relacionadas con la elección de palabras, las instrucciones sobre el uso de abreviaturas y acrónimos, y recomendaciones sobre cómo evitar el uso de nombres específicos del idioma.</span><span class="sxs-lookup"><span data-stu-id="138cb-104">This section describes general naming conventions that relate to word choice, guidelines on using abbreviations and acronyms, and recommendations on how to avoid using language-specific names.</span></span>

## <a name="word-choice"></a><span data-ttu-id="138cb-105">Elección de palabra</span><span class="sxs-lookup"><span data-stu-id="138cb-105">Word Choice</span></span>
 <span data-ttu-id="138cb-106">✔️ Elija nombres de identificador fácilmente legibles.</span><span class="sxs-lookup"><span data-stu-id="138cb-106">✔️ DO choose easily readable identifier names.</span></span>

 <span data-ttu-id="138cb-107">Por ejemplo, una propiedad denominada `HorizontalAlignment` es más legible en inglés que `AlignmentHorizontal` .</span><span class="sxs-lookup"><span data-stu-id="138cb-107">For example, a property named `HorizontalAlignment` is more English-readable than `AlignmentHorizontal`.</span></span>

 <span data-ttu-id="138cb-108">✔️ favorecer la legibilidad en lugar de la brevedad.</span><span class="sxs-lookup"><span data-stu-id="138cb-108">✔️ DO favor readability over brevity.</span></span>

 <span data-ttu-id="138cb-109">El nombre de la propiedad `CanScrollHorizontally` es mejor que `ScrollableX` (una referencia oculta al eje X).</span><span class="sxs-lookup"><span data-stu-id="138cb-109">The property name `CanScrollHorizontally` is better than `ScrollableX` (an obscure reference to the X-axis).</span></span>

 <span data-ttu-id="138cb-110">❌No use caracteres de subrayado, guiones ni cualquier otro carácter no alfanumérico.</span><span class="sxs-lookup"><span data-stu-id="138cb-110">❌ DO NOT use underscores, hyphens, or any other nonalphanumeric characters.</span></span>

 <span data-ttu-id="138cb-111">❌No use la notación húngara.</span><span class="sxs-lookup"><span data-stu-id="138cb-111">❌ DO NOT use Hungarian notation.</span></span>

 <span data-ttu-id="138cb-112">❌Evite el uso de identificadores que entren en conflicto con palabras clave de lenguajes de programación de uso generalizado.</span><span class="sxs-lookup"><span data-stu-id="138cb-112">❌ AVOID using identifiers that conflict with keywords of widely used programming languages.</span></span>

 <span data-ttu-id="138cb-113">Según la regla 4 del Common Language Specification (CLS), todos los lenguajes compatibles deben proporcionar un mecanismo que permita el acceso a los elementos con nombre que utilizan una palabra clave de ese lenguaje como identificador.</span><span class="sxs-lookup"><span data-stu-id="138cb-113">According to Rule 4 of the Common Language Specification (CLS), all compliant languages must provide a mechanism that allows access to named items that use a keyword of that language as an identifier.</span></span> <span data-ttu-id="138cb-114">C#, por ejemplo, usa el signo @ como mecanismo de escape en este caso.</span><span class="sxs-lookup"><span data-stu-id="138cb-114">C#, for example, uses the @ sign as an escape mechanism in this case.</span></span> <span data-ttu-id="138cb-115">Sin embargo, sigue siendo una buena idea evitar palabras clave comunes, ya que es mucho más difícil usar un método con la secuencia de escape que una sin ella.</span><span class="sxs-lookup"><span data-stu-id="138cb-115">However, it is still a good idea to avoid common keywords because it is much more difficult to use a method with the escape sequence than one without it.</span></span>

## <a name="using-abbreviations-and-acronyms"></a><span data-ttu-id="138cb-116">Usar abreviaturas y acrónimos</span><span class="sxs-lookup"><span data-stu-id="138cb-116">Using Abbreviations and Acronyms</span></span>
 <span data-ttu-id="138cb-117">❌No utilice abreviaturas ni contracciones como parte de los nombres de identificadores.</span><span class="sxs-lookup"><span data-stu-id="138cb-117">❌ DO NOT use abbreviations or contractions as part of identifier names.</span></span>

 <span data-ttu-id="138cb-118">Por ejemplo, use `GetWindow` en lugar de `GetWin` .</span><span class="sxs-lookup"><span data-stu-id="138cb-118">For example, use `GetWindow` rather than `GetWin`.</span></span>

 <span data-ttu-id="138cb-119">❌No use ningún acrónimo que no se acepte ampliamente y, incluso si es así, solo cuando sea necesario.</span><span class="sxs-lookup"><span data-stu-id="138cb-119">❌ DO NOT use any acronyms that are not widely accepted, and even if they are, only when necessary.</span></span>

 <span data-ttu-id="138cb-122">Por ejemplo, `GetLength` es un nombre mejor que `GetInt` .</span><span class="sxs-lookup"><span data-stu-id="138cb-122">For example, `GetLength` is a better name than `GetInt`.</span></span>

 <span data-ttu-id="138cb-123">✔️ usar un nombre de tipo CLR genérico, en lugar de un nombre específico del lenguaje, en los casos excepcionales en los que un identificador no tiene ningún significado semántico más allá de su tipo.</span><span class="sxs-lookup"><span data-stu-id="138cb-123">✔️ DO use a generic CLR type name, rather than a language-specific name, in the rare cases when an identifier has no semantic meaning beyond its type.</span></span>

 <span data-ttu-id="138cb-189">Esto ayuda a resaltar la relación entre las API.</span><span class="sxs-lookup"><span data-stu-id="138cb-189">This helps to highlight the relationship between the APIs.</span></span>

 <span data-ttu-id="138cb-191">Esto ayudará a la detección al examinar la documentación o al usar IntelliSense.</span><span class="sxs-lookup"><span data-stu-id="138cb-191">This will assist discovery when browsing documentation, or using IntelliSense.</span></span> <span data-ttu-id="138cb-192">La versión anterior de la API se organizará cerca de las nuevas API, porque la mayoría de los exploradores e IntelliSense muestran identificadores en orden alfabético.</span><span class="sxs-lookup"><span data-stu-id="138cb-192">The old version of the API will be organized close to the new APIs, because most browsers and IntelliSense show identifiers in alphabetical order.</span></span>
