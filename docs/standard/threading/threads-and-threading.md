## <a name="when-to-use-multiple-threads"></a><span data-ttu-id="85764-119">Cuándo utilizar varios subprocesos</span><span class="sxs-lookup"><span data-stu-id="85764-119">When to use multiple threads</span></span>

<span data-ttu-id="85764-120">Utilice varios subprocesos para aumentar la capacidad de respuesta de la aplicación y aprovechar las ventajas de un sistema con varios procesadores o núcleos que le permita aumentar el rendimiento de la aplicación.</span><span class="sxs-lookup"><span data-stu-id="85764-120">You use multiple threads to increase the responsiveness of your application and to take advantage of a multiprocessor or multi-core system to increase the application's throughput.</span></span>

<span data-ttu-id="85764-121">Le recomendamos que use una aplicación de escritorio en la que el subproceso principal sea responsable de los elementos de la interfaz de usuario e interactivo.</span><span class="sxs-lookup"><span data-stu-id="85764-121">Consider a desktop application, in which the primary thread is responsible for user interface elements and responds to user actions.</span></span> <span data-ttu-id="85764-122">Use subprocesos de trabajo para realizar operaciones complejas que ocuparían el subproceso principal e impedirían la interacción con la interfaz de usuario.</span><span class="sxs-lookup"><span data-stu-id="85764-122">Use worker threads to perform time-consuming operations that, otherwise, would occupy the primary thread and make the user interface non-responsive.</span></span> <span data-ttu-id="85764-123">También puede usar un subproceso dedicado para la comunicación mediante red o dispositivo, de modo que su capacidad de respuesta ante los mensajes y eventos entrantes sea mayor.</span><span class="sxs-lookup"><span data-stu-id="85764-123">You can also use a dedicated thread for network or device communication to be more responsive to incoming messages or events.</span></span>

<span data-ttu-id="85764-124">Si el programa realiza operaciones que pueden realizarse en paralelo, puede reducir el tiempo de ejecución total realizándolas en subprocesos separados y ejecutando el programa en un sistema con varios procesadores o núcleos.</span><span class="sxs-lookup"><span data-stu-id="85764-124">If your program performs operations that can be done in parallel, the total execution time can be decreased by performing those operations in separate threads and running the program on a multiprocessor or multi-core system.</span></span> <span data-ttu-id="85764-125">En un sistema de este tipo, el uso del mutithreading puede aumentar el rendimiento y la capacidad de respuesta.</span><span class="sxs-lookup"><span data-stu-id="85764-125">On such a system, use of multithreading might increase throughput along with the increased responsiveness.</span></span>
