## <a name="traditional-and-spa-behaviors-supported"></a><span data-ttu-id="5b818-142">Comportamientos tradicionales y de SPA admitidos</span><span class="sxs-lookup"><span data-stu-id="5b818-142">Traditional and SPA Behaviors Supported</span></span>

<span data-ttu-id="5b818-143">Las aplicaciones web tradicionales apenas contaban con comportamiento del lado cliente, y en su lugar se basaban en el servidor para todas las operaciones de navegación, controlador de vista de modelos (MVC) clásicos normalmente siguen este enfoque, en el que cada solicitud nueva se corresponde a otra acción de controlador, lo que a su vez podría funcionar con un <span data-ttu-id="5b818-147">Las aplicaciones de página única (SPA), por el contrario, implican muy pocas cargas de página generadas de forma dinámica en el lado de servidor (si existen).</span><span class="sxs-lookup"><span data-stu-id="5b818-147">Single Page Applications (SPAs), by contrast, involve very few dynamically generated server-side page loads (if any).</span></span> <span data-ttu-id="5b818-148">Muchas SPA se inicializan en un archivo HTML estático que carga las bibliotecas de JavaScript necesarias para iniciar y ejecutar la aplicación.</span><span class="sxs-lookup"><span data-stu-id="5b818-148">Many SPAs are initialized within a static HTML file which loads the necessary JavaScript libraries to start and run the app.</span></span> <span data-ttu-id="5b818-149">Estas aplicaciones hacen un uso intensivo de las API web para sus necesidades de datos y pueden proporcionar experiencias de usuario mucho más enriquecidas.</span><span class="sxs-lookup"><span data-stu-id="5b818-149">These apps make heavy usage of web APIs for their data needs, and can provide much richer user experiences.</span></span>

<span data-ttu-id="5b818-150">Muchas aplicaciones web implican una combinación del comportamiento de aplicación web tradicional (normalmente para el contenido) y SPA (para la interactividad).</span><span class="sxs-lookup"><span data-stu-id="5b818-150">Many web applications involve a combination of traditional web application behavior (typically for content) and SPAs (for interactivity).</span></span> <span data-ttu-id="5b818-151">ASP.NET Core admite MVC y las API web en la misma aplicación, con el mismo conjunto de herramientas y bibliotecas de marco subyacentes.</span><span class="sxs-lookup"><span data-stu-id="5b818-151">ASP.NET Core supports both MVC and web APIs in the same application, using the same set of tools and underlying framework libraries.</span></span>
