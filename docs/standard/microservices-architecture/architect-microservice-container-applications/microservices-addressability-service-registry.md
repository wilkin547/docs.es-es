# <a name="microservices-addressability-and-the-service-registry"></a><span data-ttu-id="61e5c-103">Direccionabilidad de microservicios y el Registro del servicio</span><span class="sxs-lookup"><span data-stu-id="61e5c-103">Microservices addressability and the service registry</span></span>

<span data-ttu-id="61e5c-104">Cada microservicio tiene un nombre único (URL) que se usa para resolver su ubicación.</span><span class="sxs-lookup"><span data-stu-id="61e5c-104">Each microservice has a unique name (URL) that's used to resolve its location.</span></span> <span data-ttu-id="61e5c-105">El microservicio debe ser direccionable en cualquier lugar donde se ejecute.</span><span class="sxs-lookup"><span data-stu-id="61e5c-105">Your microservice needs to be addressable wherever it's running.</span></span> <span data-ttu-id="61e5c-106">Si tiene que pensar en qué equipo se ejecuta un microservicio determinado, todo puede ir mal rápidamente.</span><span class="sxs-lookup"><span data-stu-id="61e5c-106">If you have to think about which computer is running a particular microservice, things can go bad quickly.</span></span> <span data-ttu-id="61e5c-107">De la misma manera que DNS resuelve una URL para un equipo en particular, su microservicio debe tener un nombre único para que su ubicación actual sea reconocible.</span><span class="sxs-lookup"><span data-stu-id="61e5c-107">In the same way that DNS resolves a URL to a particular computer, your microservice needs to have a unique name so that its current location is discoverable.</span></span> <span data-ttu-id="61e5c-108">Los microservicios deben tener nombres direccionables que les permitan ser independientes de la infraestructura en que se ejecutan.</span><span class="sxs-lookup"><span data-stu-id="61e5c-108">Microservices need addressable names that make them independent from the data-stu-id="61e5c-110">In the same vein, when a computer fails, the registry service must be able to indicate where the service is now running.</span></span>

<span data-ttu-id="61e5c-111">El [patrón del Registro del servicio](https://microservices.io/patterns/service-registry.html) es una parte fundamental de la detección de servicios.</span><span class="sxs-lookup"><span data-stu-id="61e5c-111">The [service registry pattern](https://microservices.io/patterns/service-registry.html) is a key part of service discovery.</span></span> <span data-ttu-id="61e5c-112">El Registro es una base de datos que contiene las ubicaciones de red de las instancias del servicio.</span><span class="sxs-lookup"><span data-stu-id="61e5c-112">The registry is a database containing the network locations of service instances.</span></span> <span data-ttu-id="61e5c-113">Un Registro del servicio debe estar muy disponible y actualizado.</span><span class="sxs-lookup"><span data-stu-id="61e5c-113">A service registry needs to be highly available and up-to-date.</span></span> <span data-ttu-id="61e5c-114">Los clientes podrían almacenar en caché las ubicaciones de red obtenidas del Registro del servicio.</span><span class="sxs-lookup"><span data-stu-id="61e5c-114">Clients could cache network locations obtained from the service registry.</span></span> <span data-ttu-id="61e5c-115">Sin embargo, esa información finalmente se queda obsoleta y los clientes ya no pueden detectar las instancias del servicio.</span><span class="sxs-lookup"><span data-stu-id="61e5c-115">However, that information eventually goes out of date and clients can no longer discover service instances.</span></span> <span data-ttu-id="61e5c-116">Por tanto, un Registro del servicio consta de un clúster de servidores que usan un protocolo de replicación para mantener su coherencia.</span><span class="sxs-lookup"><span data-stu-id="61e5c-116">Consequently, a service registry consists of a cluster of servers that use a replication protocol to maintain consistency.</span></span>

<span data-ttu-id="61e5c-117">En algunos entornos de implementación de microservicios (denominados clústeres, de los cuales se hablará en una sección posterior), la detección de servicios está integrada.</span><span class="sxs-lookup"><span data-stu-id="61e5c-117">In some microservice deployment environments (called clusters, to be covered in a later section), service discovery is built-in.</span></span> <span data-ttu-id="61e5c-118">Por ejemplo, un entorno de Azure Container Service con Kubernetes (AKS) puede controlar el Registro y la anulación del Registro de la instancia del servicio.</span><span class="sxs-lookup"><span data-stu-id="61e5c-118">For example, an Azure Container Service with Kubernetes (AKS) environment can handle service instance registration and deregistration.</span></span> <span data-ttu-id="61e5c-119">También ejecuta un proxy en cada host del clúster que desempeña el rol de enrutador de detección del lado servidor.</span><span class="sxs-lookup"><span data-stu-id="61e5c-119">It also runs a proxy on each cluster host that plays the role of server-side discovery router.</span></span> <span data-ttu-id="61e5c-120">Otro ejemplo es Azure Service Fabric, que también proporciona un Registro del servicio a través de su servicio de nombres estándar.</span><span class="sxs-lookup"><span data-stu-id="61e5c-120">Another example is Azure Service Fabric, which also provides a service registry through its out-of-the-box Naming Service.</span></span>

<span data-ttu-id="61e5c-121">Tenga en cuenta que se produce cierta superposición entre el Registro del servicio y el patrón de puerta de enlace de API, lo que también ayuda a resolver este problema.</span><span class="sxs-lookup"><span data-stu-id="61e5c-121">Note that there's certain overlap between the service registry and the API gateway pattern, which helps solve this problem as well.</span></span> <span data-ttu-id="61e5c-122">Por ejemplo, el [proxy inverso de Service Fabric](https://docs.microsoft.com/azure/service-fabric/service-fabric-reverseproxy) es un tipo de implementación de una puerta de enlace de API basada en el servicio de nombres de Service Fabric que ayuda a resolver la resolución de direcciones para los servicios internos.</span><span class="sxs-lookup"><span data-stu-id="61e5c-122">For example, the [Service Fabric Reverse Proxy](https://docs.microsoft.com/azure/service-fabric/service-fabric-reverseproxy) is a type of implementation of an API Gateway that's based on the Service Fabric Naming Service and that helps resolve address resolution to the internal services.</span></span>
