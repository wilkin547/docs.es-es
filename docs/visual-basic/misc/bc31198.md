<span data-ttu-id="dcac6-103">Una declaración literal de XML se utiliza en una expresión en una ubicación que es ambigua para el compilador de Visual Basic.</span><span class="sxs-lookup"><span data-stu-id="dcac6-103">An XML literal declaration is used in an expression in a location that is ambiguous to the Visual Basic compiler.</span></span> <span data-ttu-id="dcac6-104">Es decir, no se puede determinar el compilador de Visual Basic si el menor-que (<) del carácter está pensado como operador comparativo o para iniciar un literal XML.</span><span class="sxs-lookup"><span data-stu-id="dcac6-104">That is, the Visual Basic compiler cannot determine whether the less-than character (<) is intended as a comparative operator or the start of an XML literal.</span></span> <span data-ttu-id="dcac6-105">El siguiente fragmento de código muestra un ejemplo:</span><span class="sxs-lookup"><span data-stu-id="dcac6-105">The following code provides an example:</span></span>  

```vb  
' Generates an error.

                  Select element  
```  
  
 <span data-ttu-id="dcac6-106">**Identificador de error:** BC31198</span><span class="sxs-lookup"><span data-stu-id="dcac6-106">**Error ID:** BC31198</span></span>
