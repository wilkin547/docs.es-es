<span data-ttu-id="a933d-103">Método '\<nombredemétodo1 >' no se puede implementar el método parcial '\<nombredemétodo2 >' porque '\<nombredemétodo3 >' ya lo implementa.</span><span class="sxs-lookup"><span data-stu-id="a933d-103">Method '\<methodname1>' cannot implement partial method '\<methodname2>' because '\<methodname3>' already implements it.</span></span> <span data-ttu-id="a933d-104">Solo un método puede implementar un método parcial.</span><span class="sxs-lookup"><span data-stu-id="a933d-104">Only one method can implement a partial method.</span></span>  
  
 <span data-ttu-id="a933d-105">No es posible tener dos métodos parciales que implementan la misma declaración de método parcial.</span><span class="sxs-lookup"><span data-stu-id="a933d-105">You cannot have two partial methods that implement the same partial method declaration.</span></span> <span data-ttu-id="a933d-106">El código siguiente causa este error.</span><span class="sxs-lookup"><span data-stu-id="a933d-106">The following code causes this error.</span></span>  
  
```vb  
Partial Class Product  
  
    ' Declaration of the partial method.  
    Partial Private Sub ValueChanged()  
    End Sub  
  
End Class  
```  
  
```vb  
Partial Class Product  
  
    ' First implementation of the partial method.  
    Private Sub ValueChanged()  
        MsgBox(Value was changed to " & Me.Quantity)  
    End Sub  
  
    ' Second implementation of the partial method causes this error.  
    'Private Sub ValueChanged()  
    '    Console.WriteLine("Quantity was changed to " & Me.Quantity)  
    'End Sub  
  
End Class  
```
