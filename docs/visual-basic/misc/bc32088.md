# <a name="type-arguments-unexpected"></a><span data-ttu-id="23855-103">Argumentos de tipo no esperados</span><span class="sxs-lookup"><span data-stu-id="23855-103">Type arguments unexpected</span></span>

<span data-ttu-id="23855-104">Una cláusula `Implements` facilita argumentos de tipo para el miembro de interfaz que implementa.</span><span class="sxs-lookup"><span data-stu-id="23855-104">An `Implements` clause supplies type arguments for the interface member it is implementing.</span></span>  
  
 <span data-ttu-id="23855-105">La cláusula `Implements` solo debería identificar la interfaz y el miembro que implementa.</span><span class="sxs-lookup"><span data-stu-id="23855-105">The `Implements` clause should only identify the interface and member it is implementing.</span></span> <span data-ttu-id="23855-106">Esto significa que, si declara un procedimiento genérico, la cláusula `Of` y los argumentos de tipo deben aparecer en la parte principal de la declaración, igual que lo harían si no implementara un procedimiento de interfaz.</span><span class="sxs-lookup"><span data-stu-id="23855-106">This means that if you are declaring a generic procedure, the `Of` clause and the type arguments should appear in the main part of the declaration, just as they would if you were not implementing an interface procedure.</span></span>  
  
 <span data-ttu-id="23855-107">El código siguiente puede generar este error.</span><span class="sxs-lookup"><span data-stu-id="23855-107">The following code can generate this error.</span></span>  
  
```vb  
Public Interface testInterface  
    Sub testSub(Of t)()  
End Interface  
Public Class testClass  
    Implements testInterface  
    Public Sub testSub() Implements testInterface.testSub(Of t)()  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="23855-108">La declaración anterior a la cláusula `Implements` debe ser similar a la definición de interfaz, excepto por la posible adición de modificadores de acceso o de procedimiento.</span><span class="sxs-lookup"><span data-stu-id="23855-108">The declaration preceding the `Implements` clause should look like the interface definition, except for the possible addition of access or procedure modifiers.</span></span> <span data-ttu-id="23855-109">El código siguiente evita el error.</span><span class="sxs-lookup"><span data-stu-id="23855-109">The following code avoids the error.</span></span>  
  
```vb  
Public Sub testSub(Of t)() Implements testInterface.testSub  
```  
  
 <span data-ttu-id="23855-110">**Id. de error:** BC32088</span><span class="sxs-lookup"><span data-stu-id="23855-110">**Error ID:** BC32088</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="23855-111">Para corregir este error</span><span class="sxs-lookup"><span data-stu-id="23855-111">To correct this error</span></span>  
  
- <span data-ttu-id="23855-112">Quite la lista de argumentos de tipo de la cláusula `Implements` .</span><span class="sxs-lookup"><span data-stu-id="23855-112">Remove the type argument list from the `Implements` clause.</span></span>  
  
- <span data-ttu-id="23855-113">Si implementa un miembro genérico de la interfaz, coloque la lista de argumentos de tipo en la parte principal de la declaración, precediendo la palabra clave `Implements` .</span><span class="sxs-lookup"><span data-stu-id="23855-113">If you are implementing a generic member of the interface, then put the type argument list in the main part of the declaration, preceding the `Implements` keyword.</span></span>
