# <a name="anonymous-type-property-propertyname-cannot-be-used-in-the-definition-of-a-lambda-expression-within-the-same-initialization-list"></a><span data-ttu-id="f0bcb-103">No se puede usar la propiedad de tipo anónimo '\<propertyname>' en la definición de una expresión lambda en la misma lista de inicialización</span><span class="sxs-lookup"><span data-stu-id="f0bcb-103">Anonymous type property '\<propertyname>' cannot be used in the definition of a lambda expression within the same initialization list</span></span>

<span data-ttu-id="f0bcb-104">Las propiedades definidas en la lista de inicialización de un tipo anónimo no pueden formar parte de una definición de la expresión lambda en la misma lista.</span><span class="sxs-lookup"><span data-stu-id="f0bcb-104">Properties defined in the initialization list of an anonymous type cannot be part of a lambda expression definition in the same list.</span></span> <span data-ttu-id="f0bcb-105">Por ejemplo, en el código siguiente, la propiedad `Num` no puede incluirse en la definición de `LambdaFun`.</span><span class="sxs-lookup"><span data-stu-id="f0bcb-105">For example, in the following code, property `Num` cannot be included in the definition of `LambdaFun`.</span></span>

```vb
' Not valid.
'Dim anon = New With {.Num = 4, .LambdaFun = Function() .Num > 0}
```

<span data-ttu-id="f0bcb-106">**Identificador de error:** BC36549</span><span class="sxs-lookup"><span data-stu-id="f0bcb-106">**Error ID:** BC36549</span></span>

## <a name="to-correct-this-error"></a><span data-ttu-id="f0bcb-107">Para corregir este error</span><span class="sxs-lookup"><span data-stu-id="f0bcb-107">To correct this error</span></span>

1. <span data-ttu-id="f0bcb-108">Considere la posibilidad de dividir el tipo anónimo en dos partes:</span><span class="sxs-lookup"><span data-stu-id="f0bcb-108">Consider splitting the anonymous type into two parts:</span></span>

    ```vb
    Dim anon1 = New With {.Num = 4}
    Dim anon2 = New With {.LambdaFun = Function() anon1.Num > 0}
    ' - or -
    Dim anon3 = New With {.lambdaFun = Function(n As Integer) n > 0}
    Console.WriteLine((anon2.LambdaFun)())
    Console.WriteLine(anon3.lambdaFun(anon1.Num))
    anon1.Num = -5
    Console.WriteLine((anon2.LambdaFun)())
    Console.WriteLine(anon3.lambdaFun(anon1.Num))
    ```

    <span data-ttu-id="f0bcb-109">Tenga en cuenta que si declara `anon1.Num` como una propiedad `Key` , no se puede cambiar su valor.</span><span class="sxs-lookup"><span data-stu-id="f0bcb-109">Note that if you declare `anon1.Num` as a `Key` property, its value cannot be changed.</span></span>

2. <span data-ttu-id="f0bcb-110">Como alternativa,puede usar una instrucción de función normal para tener acceso a la propiedad de tipo anónimo:</span><span class="sxs-lookup"><span data-stu-id="f0bcb-110">An alternative is to use a regular function statement to access the anonymous type property:</span></span>

    ```vb
    Function testNum(ByVal n As Integer) As Boolean
        Return n > 0
    End Function
    Console.WriteLine(testNum(anon1.Num))
    ```

3. <span data-ttu-id="f0bcb-111">De forma similar, puede usar una función lambda definida fuera del tipo anónimo:</span><span class="sxs-lookup"><span data-stu-id="f0bcb-111">Similarly, you can use a lambda function that is defined outside the anonymous type:</span></span>

    ```vb
    Dim lambdaFun1 = Function() anon1.Num > 0
    Dim lambdaFun2 = Function(n As Integer) n > 0
    ```
