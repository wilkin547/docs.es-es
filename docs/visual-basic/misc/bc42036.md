# <a name="operands-of-type-object-used-in-expressions-for-select-case-statements-runtime-errors-could-occur"></a><span data-ttu-id="5fd1a-102">Se han utilizado operandos del tipo Object en expresiones de instrucciones 'Select', 'Case'. Podrían producirse errores en tiempo de ejecución.</span><span class="sxs-lookup"><span data-stu-id="5fd1a-102">Operands of type Object used in expressions for 'Select', 'Case' statements; runtime errors could occur</span></span>
<span data-ttu-id="5fd1a-103">Una construcción `Select`...`Case` usa una o varias expresiones del [Object Data Type](../../visual-basic/language-reference/data-types/object-data-type.md).</span><span class="sxs-lookup"><span data-stu-id="5fd1a-103">A `Select`...`Case` construction uses one or more expressions of the [Object Data Type](../../visual-basic/language-reference/data-types/object-data-type.md).</span></span>  
  
 <span data-ttu-id="5fd1a-104">Si una variable o expresión se evalúa como `Object`, el compilador debe realizar el *enlace en tiempo de ejecución*, que produce operaciones adicionales en tiempo de ejecución.</span><span class="sxs-lookup"><span data-stu-id="5fd1a-104">When a variable or expression evaluates to `Object`, the compiler must perform *late binding*, which causes extra operations at run time.</span></span> <span data-ttu-id="5fd1a-105">También expone la aplicación a posibles errores en tiempo de ejecución.</span><span class="sxs-lookup"><span data-stu-id="5fd1a-105">It also exposes your application to potential run-time errors.</span></span> <span data-ttu-id="5fd1a-106">Por ejemplo, si asigna un <xref:System.Windows.Forms.Form> a una variable `Object` e intenta usarlo para compararlo con un número, el tiempo de ejecución produce una <xref:System.InvalidCastException> porque Visual Basic no puede convertir un objeto <xref:System.Windows.Forms.Form> en un valor numérico.</span><span class="sxs-lookup"><span data-stu-id="5fd1a-106">For example, if you assign a <xref:System.Windows.Forms.Form> to an `Object` variable and then try to compare it with a number, the runtime throws an <xref:System.InvalidCastException> because Visual Basic cannot convert a <xref:System.Windows.Forms.Form> object to a numeric value.</span></span>  
  
 <span data-ttu-id="5fd1a-107">Las expresiones de una construcción `Select`...`Case` deben ser todas del mismo tipo de datos o de tipos de datos estrechamente relacionados que se puedan convertir entre sí.</span><span class="sxs-lookup"><span data-stu-id="5fd1a-107">The expressions in a `Select`...`Case` construction should all be of the same data type or of closely related data types that can be converted to each other.</span></span> <span data-ttu-id="5fd1a-108">Esto se debe a que cada instrucción `Case` compara al menos un valor con la expresión de prueba en la que se basa la construcción `Select`...`Case` .</span><span class="sxs-lookup"><span data-stu-id="5fd1a-108">This is because each `Case` statement compares at least one value against the test expression on which the `Select`...`Case` construction is based.</span></span>  
  
 <span data-ttu-id="5fd1a-109">De forma predeterminada, este mensaje es una advertencia.</span><span class="sxs-lookup"><span data-stu-id="5fd1a-109">By default, this message is a warning.</span></span> <span data-ttu-id="5fd1a-110">Para obtener más información sobre cómo ocultar las advertencias o cómo tratarlas como errores, vea [Configuring Warnings in Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).</span><span class="sxs-lookup"><span data-stu-id="5fd1a-110">For information on hiding warnings or treating warnings as errors, see [Configuring Warnings in Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).</span></span>
