# <a name="differences-between-passing-an-argument-by-value-and-by-reference-visual-basic"></a><span data-ttu-id="50b23-102">Diferencias entre pasar un argumento por valor y por referencia (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="50b23-102">Differences Between Passing an Argument By Value and By Reference (Visual Basic)</span></span>
<span data-ttu-id="50b23-103">Al pasar uno o más argumentos a un procedimiento, cada argumento corresponde a un elemento de programación subyacente en el código de llamada.</span><span class="sxs-lookup"><span data-stu-id="50b23-103">When you pass one or more arguments to a procedure, each argument corresponds to an underlying programming element in the calling code.</span></span> <span data-ttu-id="50b23-104">Puede pasar el valor de este elemento subyacente, o bien una referencia a él.</span><span class="sxs-lookup"><span data-stu-id="50b23-104">You can pass either the value of this underlying element, or a reference to it.</span></span> <span data-ttu-id="50b23-105">Esto se conoce como el *mecanismo para pasar argumentos*.</span><span class="sxs-lookup"><span data-stu-id="50b23-105">This is known as the *passing mechanism*.</span></span>  
  
## <a name="passing-by-value"></a><span data-ttu-id="50b23-106">Pasar por valor</span><span class="sxs-lookup"><span data-stu-id="50b23-106">Passing by Value</span></span>  
 <span data-ttu-id="50b23-107">Se pasa un argumento *por valor* especificando la [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) palabra clave para el parámetro correspondiente en la definición del procedimiento.</span><span class="sxs-lookup"><span data-stu-id="50b23-107">You pass an argument *by value* by specifying the [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="50b23-108">Cuando se utiliza este mecanismo para pasar argumentos, Visual Basic copia el valor del elemento de programación subyacente en una variable local en el procedimiento.</span><span class="sxs-lookup"><span data-stu-id="50b23-108">When you use this passing mechanism, Visual Basic copies the value of the underlying programming element into a local variable in the procedure.</span></span> <span data-ttu-id="50b23-109">El código del procedimiento no tiene acceso al elemento subyacente en el código de llamada.</span><span class="sxs-lookup"><span data-stu-id="50b23-109">The procedure code does not have any access to the underlying element in the calling code.</span></span>  
  
## <a name="passing-by-reference"></a><span data-ttu-id="50b23-110">Pasar por referencia</span><span class="sxs-lookup"><span data-stu-id="50b23-110">Passing by Reference</span></span>  
 <span data-ttu-id="50b23-111">Se pasa un argumento *por referencia* especificando la [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) palabra clave para el parámetro correspondiente en la definición del procedimiento.</span><span class="sxs-lookup"><span data-stu-id="50b23-111">You pass an argument *by reference* by specifying the [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="50b23-112">Cuando se utiliza este mecanismo para pasar argumentos, Visual Basic proporciona el procedimiento una referencia directa al elemento de programación subyacente en el código de llamada.</span><span class="sxs-lookup"><span data-stu-id="50b23-112">When you use this passing mechanism, Visual Basic gives the procedure a direct reference to the underlying programming element in the calling code.</span></span>  
  
## <a name="passing-mechanism-and-element-type"></a><span data-ttu-id="50b23-113">Mecanismo para pasar argumentos y el tipo de elemento</span><span class="sxs-lookup"><span data-stu-id="50b23-113">Passing Mechanism and Element Type</span></span>  
 <span data-ttu-id="50b23-114">La elección del mecanismo para pasar argumentos no es igual que la clasificación del tipo de elemento subyacente.</span><span class="sxs-lookup"><span data-stu-id="50b23-114">The choice of passing mechanism is not the same as the classification of the underlying element type.</span></span> <span data-ttu-id="50b23-115">Pasar por valor o por referencia hace referencia a lo que proporciona Visual Basic en el código de procedimiento.</span><span class="sxs-lookup"><span data-stu-id="50b23-115">Passing by value or by reference refers to what Visual Basic supplies to the procedure code.</span></span> <span data-ttu-id="50b23-116">Un tipo de valor o tipo de referencia hace referencia a cómo se almacena un elemento de programación en la memoria.</span><span class="sxs-lookup"><span data-stu-id="50b23-116">A value type or reference type refers to how a programming element is stored in memory.</span></span>  
  
 <span data-ttu-id="50b23-117">Sin embargo, el mecanismo para pasar argumentos y el tipo de elemento están relacionados entre sí.</span><span class="sxs-lookup"><span data-stu-id="50b23-117">However, the passing mechanism and element type are interrelated.</span></span> <span data-ttu-id="50b23-118">El valor de un tipo de referencia es un puntero a los datos en otra ubicación en la memoria.</span><span class="sxs-lookup"><span data-stu-id="50b23-118">The value of a reference type is a pointer to the data elsewhere in memory.</span></span> <span data-ttu-id="50b23-119">Esto significa que, cuando se pasa un tipo de referencia por valor, el código de procedimiento tiene un puntero a datos del elemento subyacente, aunque no tiene acceso al propio elemento subyacente.</span><span class="sxs-lookup"><span data-stu-id="50b23-119">This means that when you pass a reference type by value, the procedure code has a pointer to the underlying element's data, even though it cannot access the underlying element itself.</span></span> <span data-ttu-id="50b23-120">Por ejemplo, si el elemento es una variable de matriz, el código del procedimiento no tiene acceso a la variable en Sí, pero pueden tener acceso a los miembros de la matriz.</span><span class="sxs-lookup"><span data-stu-id="50b23-120">For example, if the element is an array variable, the procedure code does not have access to the variable itself, but it can access the array members.</span></span>
