## <a name="container-design-principles"></a><span data-ttu-id="1388c-113">Principios de diseño de contenedores</span><span class="sxs-lookup"><span data-stu-id="1388c-113">Container design principles</span></span>

<span data-ttu-id="1388c-114">En el modelo de contenedor, una instancia de imagen de contenedor representa un único proceso.</span><span class="sxs-lookup"><span data-stu-id="1388c-114">In the container model, a container image instance represents a single process.</span></span> <span data-ttu-id="1388c-115">Al definir una imagen de contenedor como un límite de proceso, puede crear primitivas que se usen para escalar el proceso o para procesarlo por lotes.</span><span class="sxs-lookup"><span data-stu-id="1388c-115">By defining a container image as a process boundary, you can create primitives that can be used to scale the process or to batch it.</span></span>

<span data-ttu-id="1388c-116">Al diseñar una imagen de contenedor, verá una definición [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/#entrypoint) en el archivo Dockerfile.</span><span class="sxs-lookup"><span data-stu-id="1388c-116">When you design a container image, you'll see an [ENTRYPOINT](https://docs.docker.com/engine/reference/builder/#entrypoint) definition in the Dockerfile.</span></span> <span data-ttu-id="1388c-117">Esto define el proceso cuya duración controla la duración del contenedor.</span><span class="sxs-lookup"><span data-stu-id="1388c-117">This definition defines the process whose lifetime controls the lifetime of the container.</span></span> <span data-ttu-id="1388c-118">Cuando se completa el proceso, finaliza el ciclo de vida del contenedor.</span><span class="sxs-lookup"><span data-stu-id="1388c-118">When the process completes, the container lifecycle ends.</span></span> <span data-ttu-id="1388c-119">Los contenedores pueden representar procesos de ejecución prolongada como servidores web, pero también pueden representar procesos de corta duración, como trabajos por lotes, que anteriormente se implementarían como [WebJobs](https://github.com/Azure/azure-webjobs-sdk/wiki) de Azure.</span><span class="sxs-lookup"><span data-stu-id="1388c-119">Containers might represent long-running processes like web servers, but can also represent short-lived processes like batch jobs, which formerly might have been implemented as Azure [WebJobs](https://github.com/Azure/azure-webjobs-sdk/wiki).</span></span>

<span data-ttu-id="1388c-120">Si se produce un error en el proceso, el contenedor finaliza y lo sustituye el orquestador.</span><span class="sxs-lookup"><span data-stu-id="1388c-120">If the process fails, the container ends, and the orchestrator takes over.</span></span> <span data-ttu-id="1388c-121">Si el orquestador está configurado para mantener cinco instancias en ejecución y se produce un error en una de ellas, el orquestador creará otra instancia del contenedor para reemplazar al proceso con error.</span><span class="sxs-lookup"><span data-stu-id="1388c-121">If the orchestrator was configured to keep five instances running and one fails, the orchestrator will create another container instance to replace the failed process.</span></span> <span data-ttu-id="1388c-122">En un trabajo por lotes, el proceso se inicia con parámetros.</span><span class="sxs-lookup"><span data-stu-id="1388c-122">In a batch job, the process is started with parameters.</span></span> <span data-ttu-id="1388c-123">Cuando el proceso finalice, el trabajo se habrá completado.</span><span class="sxs-lookup"><span data-stu-id="1388c-123">When the process completes, the work is complete.</span></span> <span data-ttu-id="1388c-124">Más adelante en esta guía se exploran en profundidad los orquestadores.</span><span class="sxs-lookup"><span data-stu-id="1388c-124">This guidance drills-down on orchestrators, later on.</span></span>

<span data-ttu-id="1388c-125">Es posible que en algún momento le interese que varios procesos se ejecuten en un solo contenedor.</span><span class="sxs-lookup"><span data-stu-id="1388c-125">You might find a scenario where you want multiple processes running in a single container.</span></span> <span data-ttu-id="1388c-126">En ese caso, dado que solo puede haber un punto de entrada por contenedor, puede ejecutar dentro del contenedor un script que inicie todos los programas que sean necesarios.</span><span class="sxs-lookup"><span data-stu-id="1388c-126">For that scenario, since there can be only one entry point per container, you could run a script within the container that launches as many programs as needed.</span></span> <span data-ttu-id="1388c-127">Por ejemplo, puede usar [Supervisor](http://supervisord.org/) o una herramienta similar para que se encargue de iniciar varios procesos dentro de un único contenedor.</span><span class="sxs-lookup"><span data-stu-id="1388c-127">For example, you can use [Supervisor](http://supervisord.org/) or a similar tool to take care of launching multiple processes inside a single container.</span></span> <span data-ttu-id="1388c-128">Este enfoque no es muy habitual, aunque existan arquitecturas que contengan varios procesos por contenedor.</span><span class="sxs-lookup"><span data-stu-id="1388c-128">However, even though you can find architectures that hold multiple processes per container, that approach isn't very common.</span></span>
