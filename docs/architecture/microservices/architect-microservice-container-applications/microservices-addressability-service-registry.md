class="sxs-lookup"><span data-stu-id="67d00-105">Your microservice needs to be addressable wherever it's running.</span></span> <span data-ttu-id="67d00-106">Si tiene que pensar en qué equipo se ejecuta un microservicio determinado, todo puede ir mal rápidamente.</span><span class="sxs-lookup"><span data-stu-id="67d00-106">If you have to think about which computer is running a particular microservice, things can go bad quickly.</span></span> <span data-ttu-id="67d00-107">De la misma manera que DNS resuelve una URL para un equipo en particular, su microservicio debe tener un nombre único para que su ubicación actual sea reconocible.</span><span class="sxs-lookup"><span data-stu-id="67d00-107">In the same way that DNS resolves a URL to a particular computer, your microservice <span data-ttu-id="67d00-117">En algunos entornos de implementación de microservicios (denominados clústeres, de los cuales se hablará en una sección posterior), la detección de servicios está integrada.</span><span class="sxs-lookup"><span data-stu-id="67d00-117">In some microservice deployment environments (called clusters, to be covered in a later section), service discovery is built in.</span></span> <span data-ttu-id="67d00-118">Por ejemplo, un entorno de Azure Container Service con Kubernetes (AKS) puede controlar el Registro y la anulación del Registro de la instancia del servicio.</span><span class="sxs-lookup"><span data-stu-id="67d00-118">For example, an Azure Container Service with Kubernetes (AKS) environment can handle service instance registration and deregistration.</span></span> <span data-ttu-id="67d00-119">También ejecuta un proxy en cada host del clúster que desempeña el rol de enrutador de detección del lado servidor.</span><span class="sxs-lookup"><span data-stu-id="67d00-119">It also runs a proxy on each cluster host that plays the role of server-side discovery router.</span></span>
