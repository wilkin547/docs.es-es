## <a name="prepare-the-data"></a><span data-ttu-id="b88da-135">Preparación de los datos</span><span class="sxs-lookup"><span data-stu-id="b88da-135">Prepare the data</span></span>

       .Read()
       .Schema("id INT, url STRING, owner_id INT, " +

   ```

1. <span data-ttu-id="b88da-140">Use el método <xref:Microsoft.Spark.Sql.DataFrame.Na%2A> para colocar filas con valores NA (NULL) y el método <xref:Microsoft.Spark.Sql.DataFrame.Drop%2A> para quitar ciertas columnas de los datos.</span><span class="sxs-lookup"><span data-stu-id="b88da-140">Use the <xref:Microsoft.Spark.Sql.DataFrame.Na%2A> method to drop rows with NA (null) values, and the <xref:Microsoft.Spark.Sql.DataFrame.Drop%2A> method to remove certain columns from your data.</span></span> <span data-ttu-id="b88da-141">Esto ayuda a evitar errores si intenta analizar datos nulos o columnas que no son de interés para el análisis final.</span><span class="sxs-lookup"><span data-stu-id="b88da-141">This helps prevent errors if you try to analyze null data or columns that are not relevant to your final analysis.</span></span>

   DataFrameNaFunctions dropEmptyProjects = projectsDf.Na();
   DataFrame cleanedProjects = dropEmptyProjects.Drop("any");

   // Remove unnecessary columns
   cleanedProjects = cleanedProjects.Drop("id", "url", "owner_id");
   cleanedProjects.Show();
   ```

## <a name="analyze-the-data"></a><span data-ttu-id="b88da-142">Análisis de los datos</span><span class="sxs-lookup"><span data-stu-id="b88da-142">Analyze the data</span></span>

<span data-ttu-id="b88da-147">El objetivo de esta aplicación es obtener información detallada sobre los datos de los proyectos de GitHub.</span><span class="sxs-lookup"><span data-stu-id="b88da-147">The goal of this app is to gain some insights about the GitHub projects data.</span></span> <span data-ttu-id="b88da-148">Agregue los siguientes fragmentos de código al programa para analizar los datos.</span><span class="sxs-lookup"><span data-stu-id="b88da-148">Add the following code snippets to your program to analyze the data.</span></span>

1. <span data-ttu-id="b88da-149">Al agregar el siguiente bloque de código se busca el número de veces que se ha bifurcado cada idioma.</span><span class="sxs-lookup"><span data-stu-id="b88da-149">Add the following block of code finds the number of times each language has been forked.</span></span> <span data-ttu-id="b88da-150">En primer lugar, los datos se agrupan por idioma.</span><span class="sxs-lookup"><span data-stu-id="b88da-150">First, the data is grouped by language.</span></span> <span data-ttu-id="b88da-151">A continuación, se toma el número promedio de bifurcaciones de cada idioma.</span><span class="sxs-lookup"><span data-stu-id="b88da-151">Then, the average number of forks from each language is taken.</span></span>

   ```csharp
   // Average number of times each language has been forked
   DataFrame groupedDF = cleanedProjects
       .GroupBy("language")
       .Agg(Avg(cleanedProjects["forked_from"]);
   ```

1. <span data-ttu-id="b88da-152">Agregue el siguiente bloque de código para ordenar el promedio de bifurcaciones en orden descendente para ver los idiomas más bifurcados.</span><span class="sxs-lookup"><span data-stu-id="b88da-152">Add the following block of code to order the average number of forks in descending order to see which languages are the most forked.</span></span> <span data-ttu-id="b88da-153">Es decir, el mayor número de bifurcaciones aparecerá en primer lugar.</span><span class="sxs-lookup"><span data-stu-id="b88da-153">That is, the largest number of forks will appear first.</span></span>

   ```csharp
   // Sort by most forked languages first
   groupedDF.OrderBy(Desc("avg(forked_from)")).Show();
   ```

1. <span data-ttu-id="b88da-154">El siguiente bloque de código muestra cómo de recientemente se han actualizado los proyectos.</span><span class="sxs-lookup"><span data-stu-id="b88da-154">The next block of code shows you how recently projects have been updated.</span></span> <span data-ttu-id="b88da-155">Registre una nueva función definida por el usuario llamada *MyUDF* y compárela con una fecha, *s_referenceDate*, que se declaró al principio del tutorial.</span><span class="sxs-lookup"><span data-stu-id="b88da-155">You register a new user-defined function called *MyUDF* and compare it with a date, *s_referenceDate*, which was declared at the beginning of the tutorial.</span></span> <span data-ttu-id="b88da-156">La fecha de cada proyecto se compara con la fecha de referencia.</span><span class="sxs-lookup"><span data-stu-id="b88da-156">The date for each project is compared against the reference date.</span></span> <span data-ttu-id="b88da-157">A continuación, Spark SQL se usa para llamar a la función definida por el usuario en cada fila de los datos para analizar cada proyecto del conjunto de datos.</span><span class="sxs-lookup"><span data-stu-id="b88da-157">Then, Spark SQL is used to call the UDF on each row of the data to analyze each project in the data set.</span></span>

   ```csharp
   spark.Udf().Register<string, bool>(
       "MyUDF",
       (date) => DateTime.TryParse(date, out DateTime convertedDate) &&
           (convertedDate > s_referenceDate);
   cleanedProjects.CreateOrReplaceTempView("dateView");

   DataFrame dateDf = spark.Sql(
       "SELECT *, MyUDF(dateView.updated_at) AS datebefore FROM dateView");
   dateDf.Show();
   ```
